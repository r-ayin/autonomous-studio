## Derived Research Fixes（applicability=high）

以下发现转入 `audit-cycle-state.derived_fixes`，kind=research-informed，后续派发轮走正常 fix 管线。

### RF-001: SenseVoice 替换 faster-whisper 获得情绪+语种多维信号
- **gap**: `asr.py:82-95` FasterWhisperTranscriber 仅产出 text+timestamp，无情绪/语种；`distill.py` 情绪推断靠关键词匹配（`llm.py:105-126` emo_kw 列表），漏掉声学情绪
- **reference**: SenseVoice (FunAudioLLM) — 单次前向 ASR+SER+LID+audio event
- **remediation sketch**: 新增 `SenseVoiceTranscriber(Transcriber)` backend；Segment dataclass 加 `emotion: str = ""` 和 `lang_detected: str = ""` 字段；storage schema migration；config 加 `asr.sense_voice` 节
- **effort**: M
- **source**: {type: oss, name: SenseVoice, url: https://github.com/FunAudioLLM/SenseVoice, reference_idea: single-forward multi-task ASR+SER+LID}

### RF-002: 引入分层记忆架构（Letta/MemGPT 风格）替代全量塞 prompt
- **gap**: `memory.py` + `distill.py` 产出的人格档案全量注入 system prompt；记忆量增长后要么 prompt 爆炸要么被迫截断丢失信息；无语义检索
- **reference**: Letta (MemGPT) — core/archival/recall 三层 + agent 自编辑
- **remediation sketch**: 定义 MemoryTier 抽象（working/long_term/episodic）；working = 当前会话 context；long_term = 向量索引 + 元数据；episodic = 时间线索引；加 memory_search / memory_edit tool 让 LLM 自主管理
- **effort**: L
- **source**: {type: oss, name: Letta, url: https://github.com/letta-ai/letta, reference_idea: tiered self-editing memory with agent-controlled paging}

### RF-003: 接入真实向量检索替代 HashingEmbedder
- **gap**: `llm.py:475-497` HashingEmbedder 是确定性哈希投影，无语义理解能力；记忆召回靠 keyword/DuckDB SQL，无法做"找类似经历"的语义搜索
- **reference**: Mem0 extract-and-retrieve + Letta archival memory 均依赖真实 embedding
- **remediation sketch**: OpenAICompatEmbedder 已存在（`llm.py:500-513`）但未用于记忆检索；加 sqlite-vss 或 chromadb 本地向量索引；recall 路径改为 embed query → top-k 相似记忆 → 注入 prompt
- **effort**: M
- **source**: {type: oss, name: Mem0, url: https://github.com/mem0ai/mem0, reference_idea: automatic fact extraction + vector retrieval for conversational memory}
