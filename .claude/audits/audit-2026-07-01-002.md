## Findings

### H-001 [HIGH] worktree add 失败后残留空目录 + 后续操作在非 worktree 上执行
- **File**: scripts/opt-worktree.sh:327, 348
- **Kind**: route-fix
- **Evidence**: `git -C "$PROJECT" worktree add -b "$new_wt_branch" "$target" "$MAIN_BRANCH" 2>/dev/null` 失败时 stderr 被吞；line 326 `mkdir -p "$target"` 已建出空 dir；set -e 在复合命令中行为不可靠
- **Reproduction**: MAIN_BRANCH ref 不存在或分支名已存在 → worktree add exit 128 → set -e 可能 abort 但 $target 空目录残留
- **Remediation**: 仿 ensure_main_wt 模式：worktree add 失败时 rmdir $target + 明确 exit 1

### H-005 [HIGH] claude 输出全量捕获到 shell 变量致内存膨胀 + 敏感信息暴露
- **File**: scripts/autonomous-loop.sh:73
- **Kind**: route-fix
- **Evidence**: `out=$(claude -p "$PROMPT" ... 2>&1)` 后 `echo "$out" | tail -30` + `echo "$out" | grep -qiE '402|...'` 三次复制 MB 级字符串
- **Reproduction**: claude -p 长时间运行产出大量输出（含文件内容、工具结果）→ bash 变量膨胀 + pipe buffer 含全部敏感信息
- **Remediation**: 写到 mktemp 临时文件，tail/grep 直接读文件；结束后 shred/rm
