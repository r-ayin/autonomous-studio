## 项目概况

wechat-main 是微信公众号深度长文生产管线：选题扫描 → godtier 深度研究 → persona 人格化重写 → QA 门禁 → 输出。技术栈纯 Python stdlib + urllib，无第三方依赖；外部调用包括邮件/QQ Bot 推送、Bocha/Brave/Bing 搜索 API、DeepSeek LLM API。凭证全部 env 读取（`.wechat-env` gitignored），TLS 默认开启，HTTP 超时齐全。整体代码质量较高，但存在若干安全与健壮性问题。

## HIGH Findings (5)

### H-001 research_cache.py:115 — slug path traversal (R/W arbitrary files)
- **file**: scripts/research_cache.py:115, 165, 184
- **severity**: high
- **finding**: `slug` CLI arg flows unsanitized into `f"{slug}.json"` path construction across get/put/diff subcommands. A slug like `../../etc/cron.d/malicious` reads or writes arbitrary files.
- **evidence**: `cache_path = _CACHE_DIR / f"{slug}.json"` — slug from argparse with no regex/validation.
- **remediation**: Validate slug against `re.fullmatch(r'[a-z0-9][a-z0-9_-]{0,63}', slug)`. Reject `/`, `\`, `..`.
- **kind**: route-fix

### H-002 pipeline.py:222 — shell injection via state['min_bytes'] f-string interpolation
- **file**: scripts/pipeline.py:222
- **severity**: high
- **finding**: `cmd = f"WECHAT_MIN_BYTES={state['min_bytes']} {cmd}"` prepends state value to shell command. State file is JSON loaded without schema validation (see M-010); attacker-controllable state can inject shell metacharacters.
- **evidence**: Line 222 interpolates state['min_bytes'] directly; _load() at line 64 trusts arbitrary JSON.
- **remediation**: Pass WECHAT_MIN_BYTES via subprocess env dict (as already done in _run_gate) instead of shell prefix. Or int-cast + validate before interpolation.
- **kind**: route-fix

### H-003 bing_search.py:153 — SSRF via _fetch_page_text arbitrary URL fetch
- **file**: scripts/bing_search.py:153-197
- **severity**: high
- **finding**: `_fetch_page_text(url)` fetches ANY URL extracted from Bing SERP results without protocol/host/IP validation. Attacker-controlled SERP entry can target internal services (169.254.169.254, 10.x, localhost). Redirects followed by default, post-redirect destination unchecked.
- **evidence**: `urllib.request.urlopen(req, timeout=timeout)` on unvalidated URL; `_SKIP_DOMAINS` checked only pre-fetch.
- **remediation**: Allow only http/https; block private/reserved IP ranges (use `ipaddress` module); re-validate after redirect resolution.
- **kind**: route-fix

### H-004 fact_checker.py:322 — empty search results auto-FALSIFIED enables search-poisoning DoS
- **file**: script-verifier/fact_checker.py:322-334
- **severity**: high
- **finding**: When search_result is empty and claim.risk=='high', verdict is unconditionally FALSIFIED. Adversary who influences search results (SEO poisoning) can force any precise-data claim to be marked FALSIFIED by ensuring no corroborating results appear. Combined with 3-strike remediation loop, forces deletion of factual content or infinite retry.
- **evidence**: Lines 330-334: `if not search_result and claim.risk == 'high': verdict = 'FALSIFIED'`.
- **remediation**: Treat empty results as UNVERIFIABLE with needs_claude_review=True. Require affirmative contradicting evidence for FALSIFIED. Add manual-review escape hatch.
- **kind**: route-fix

### H-005 qq_push.py:42 — hardcoded /opt/wanxia/.env path with no permission check
- **file**: scripts/qq_push.py:42
- **severity**: high
- **finding**: `_DEFAULT_ENV = "/opt/wanxia/.env"` hardcoded absolute path. File permissions not checked; if world-readable, QQ Bot ClientSecret leaks to any local user.
- **evidence**: Lines 42-48: reads file without stat mode check.
- **remediation**: On load, stat-check file mode; reject if not 0o600/0o400. Prefer env var over .env fallback. Document chmod requirement.
- **kind**: route-fix
