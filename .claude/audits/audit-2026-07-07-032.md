### M-POST-002 — creds.js /proc/${pid}/environ read uses unvalidated PID string

- **Severity**: medium
- **Kind**: route-fix
- **File**: `runner-exec/scripts/creds.js:20-24`
- **Evidence**: Line 20: `const pid = execFileSync('pgrep', ['-f', 'node.*cloud-cli.*server'], ...).trim().split('\n')[0];` — pgrep may return empty string if no match, or a non-numeric string if pgrep outputs warnings to stdout. Line 22: `fs.readFileSync(\`/proc/${pid}/environ\`, 'utf-8')` interpolates pid directly into path without numeric validation. Empty pid → reads `/proc//environ` (EINVAL, caught by catch). Non-numeric pid → reads arbitrary `/proc/<string>/environ` (ENOENT, caught). No TOCTOU risk since /proc is kernel-managed, but the pattern is fragile.
- **Impact**: Low practical exploitability (catch block swallows errors), but violates defense-in-depth. A future refactor removing the catch would expose path traversal.
- **Remediation**: Add `if (!/^\d+$/.test(pid)) throw new Error('invalid pid');` before the readFileSync call. Single-line fix in creds.js.

### L-POST-001 — WebSocket auth token in URL query string (log/proxy exposure)

- **Severity**: low
- **Kind**: route-fix
- **Files**: `runner-exec/scripts/exec.js:16`, `runner-exec/scripts/session.js:22`, `runner-exec/scripts/write-file.js:28`, `runner-exec/scripts/batch-write.js:17`
- **Evidence**: All 4 files construct WebSocket URL as `${WS_HOST}/runner-shell/${runnerId}?token=${token}`. Session tokens in URL query parameters are captured by: (1) proxy access logs, (2) browser history (if applicable), (3) network monitoring tools, (4) error reporting that includes URLs.
- **Impact**: Token leakage via log aggregation. The token grants full PowerShell access to the Runner. Mitigated by: localhost-only WS connection (WS_HOST derived from 127.0.0.1:58596 default), short-lived sessions, internal-use-only tool.
- **Remediation**: Move token to WebSocket subprotocol header or first-message authentication handshake. Requires coordinated change with cloud-cli server endpoint — direction-shift scope. Low priority given localhost binding.
