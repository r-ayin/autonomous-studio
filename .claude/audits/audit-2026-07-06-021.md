## Findings

### H-001: /ws/xiaozhi endpoint skips WebSocket token authentication entirely
- **File**: `personal_assistant/api.py:199`
- **Severity**: high
- **Kind**: route-fix
- **Evidence**: `async def ws_xiaozhi(ws): await xiaozhi_server.xiaozhi_endpoint(ws)` — no `auth.verify_ws_token(ws)` check, unlike `/ws/live` (line 131) and `/ws/audio` (line 175) which both verify tokens before proceeding.
- **Remediation**: Add `if not auth.verify_ws_token(ws): await ws.close(code=1008); return` before calling `xiaozhi_server.xiaozhi_endpoint(ws)`, matching the pattern in `ws_live` and `ws_audio`.

### H-002: CORS wildcard + dev-mode auth bypass enables cross-origin data exfiltration
- **File**: `personal_assistant/api.py:106`
- **Severity**: high
- **Kind**: route-fix
- **Evidence**: `app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])` combined with `auth.py:30`: `if not expected: return True` (dev mode passthrough when `PA_API_TOKEN` unset). Any webpage can make authenticated requests if token happens to be set but origin is unrestricted.
- **Remediation**: Replace `allow_origins=['*']` with explicit trusted origin list from config; at minimum restrict to same-origin when auth is enabled.

### H-003: WebSocket relay token leaked in plaintext to stdout log
- **File**: `scripts/ecs_relay/tunnel_client.py:51`
- **Severity**: high
- **Kind**: route-fix
- **Evidence**: `print(f"[tunnel] 连 ECS {ecs_url}", flush=True)` where `ecs_url = f'wss://...:{ECS_PORT}/ws/tunnel{token_qs}'` and `token_qs = f'?token={RELAY_TOKEN}'`. Token appears in process stdout, captured by supervisord/journalctl/log aggregators.
- **Remediation**: Log URL with token redacted: replace `token_qs` with `'?token=***'` in print statement, or log only host/port/path without query string.

### H-004: Audio relay WebSocket binds 0.0.0.0 with no authentication
- **File**: `scripts/audio_relay.py:222`
- **Severity**: high
- **Kind**: route-fix
- **Evidence**: `async with websockets.serve(self.handler, "0.0.0.0", self.port)` — no token check in `handler()`, no origin validation, no per-IP rate limits. Any network peer can inject PCM audio into the assistant pipeline.
- **Remediation**: Bind to `127.0.0.1` if ESP32 is local; otherwise add token verification in `handler()` mirroring `relay_bridge._check_token` pattern, plus `max_size` + rate limiting.

### M-001: /ws/ prefix exempted from HTTP auth middleware but new WS endpoints inherit no auth
- **File**: `personal_assistant/auth.py:49`
- **Severity**: medium
- **Kind**: route-fix
- **Evidence**: `_EXEMPT_PREFIXES = ("/health", "/web", "/android", "/favicon", "/ws/")` exempts all `/ws/*` from HTTP middleware; `ws_xiaozhi` at `api.py:199` has no `verify_ws_token` call. Comment says "WS upgrade bypasses HTTP middleware anyway" but this means each WS handler must independently authenticate — and `ws_xiaozhi` doesn't.
- **Remediation**: Remove `/ws/` from `_EXEMPT_PREFIXES` (document that WS handlers must self-authenticate); audit all `@app.websocket` handlers to ensure each calls `verify_ws_token`.

### M-002: POST /chat blocks async event loop with synchronous LLM call
- **File**: `personal_assistant/api.py:325`
- **Severity**: medium
- **Kind**: route-fix
- **Evidence**: `@app.post("/chat") def converse(body: ChatIn): reply, evidence = chat.Assistant().respond(body.message)` — sync blocking call in async framework. Unlike `ws_live` which uses `asyncio.to_thread`, this blocks the entire event loop during LLM inference (seconds), causing WebSocket heartbeat timeouts and potential DoS.
- **Remediation**: Convert to `async def` and wrap with `await asyncio.to_thread(chat.Assistant().respond, body.message)`, matching the pattern in `_handle_live_message` at line 161.

### M-003: Memory/event/reminder IDs derived from Python hash() are predictable and collision-prone
- **File**: `personal_assistant/storage.py:88`
- **Severity**: medium
- **Kind**: route-fix
- **Evidence**: `mid = mem.get('id') or f"m-{abs(hash((mem.get('segment_id',''), mem.get('content','')[:40])))%10**12}"` — Python `hash()` is deterministic per-process for strings; truncating content to 40 chars increases collisions. `INSERT OR REPLACE` then silently overwrites existing records.
- **Remediation**: Use `secrets.token_hex(8)` or `uuid4` for ID generation; or use content-addressable SHA-256 if determinism is required.

### M-004: .env parser silently mishandles values containing '=' signs and mixed quoting
- **File**: `personal_assistant/config.py:17`
- **Severity**: medium
- **Kind**: route-fix
- **Evidence**: `k, _, v = line.partition("=")` correctly splits on first `=`, but `v.strip().strip('"').strip("'")` strips quotes asymmetrically — a value like `"key=with=equals"` works, but mixed quoting like `"it's"` leaves residual quote chars, corrupting secrets like base64-encoded API keys.
- **Remediation**: Use `shlex.split` or a proper `.env` parser (e.g., `python-dotenv`); at minimum handle quoted values by checking if `v` starts and ends with matching quotes before stripping.

### M-005: Origin validation bypassed when Origin header is absent
- **File**: `scripts/ecs_relay/relay_bridge.py:107`
- **Severity**: medium
- **Kind**: route-fix
- **Evidence**: `if not origin: return True  # 非浏览器客户端不发 Origin，放行` — an attacker can simply omit the Origin header to skip the allowlist check entirely. Token is the stated primary defense, but defense-in-depth requires both.
- **Remediation**: For browser-initiated connections (detected via User-Agent or Sec-WebSocket-* headers), require Origin; only allow missing Origin for known non-browser client patterns, or require token+origin together.

### M-006: inbox/upload accepts arbitrary request body size with no limit
- **File**: `personal_assistant/api.py:483`
- **Severity**: medium
- **Kind**: route-fix
- **Evidence**: `content = await request.body()` — no size check before reading entire body into memory and writing to disk via `dest.write_bytes(content)`. Enables disk exhaustion DoS.
- **Remediation**: Add max body size check: stream-read with limit and reject if exceeding configurable threshold (e.g., 100MB for audio files) before writing to disk.

### L-001: Heartbeat sends zero-byte binary frame instead of WebSocket ping
- **File**: `personal_assistant/ws_manager.py:91`
- **Severity**: low
- **Kind**: route-fix
- **Evidence**: `await ws.send_bytes(b"")  # 零字节探活；uvicorn 会发 pong` — `send_bytes(b'')` is a data frame, not a control ping frame. Behavior depends on uvicorn internals and may not trigger pong on all clients.
- **Remediation**: Use protocol-level ping (`await ws.ping()`) or rely on framework built-in keepalive.

### L-002: POST /verify returns assertion failures in 200 response body, masking from monitoring
- **File**: `personal_assistant/api.py:339`
- **Severity**: low
- **Kind**: route-fix
- **Evidence**: `except AssertionError as e:` catches test failures and returns them as string in 200 response body rather than HTTP error status, making regressions invisible to monitoring/alerting systems.
- **Remediation**: Return HTTP 500 or dedicated error status when assertion fails so monitoring can detect regressions.
