## Findings

### GD-H-001 · high · correctness/silent-failure · route-fix
- **file**: scripts/computation/utils/audit_log.py
- **line**: 14（`"input_hash": hash(str(inputs))[:16] if inputs else None`）
- **finding**: `hash()` 返回 **int**，`int[:16]` → `TypeError: 'int' object is not subscriptable`。每当 `inputs` 真值（非空/非零）时，`log()` 立即抛 TypeError。**整个计算审计日志通道在每次真实调用时崩溃**——而审计日志正是本 skill 给所有计算操作留痕的唯一机制（等价 B 节 audit-log instrumentation）。下游若不裸 `except` 吞错，计算脚本会因记日志而失败；若吞错，则审计 trail 静默丢失。两种结局都使审计目的落空。
- **evidence**:
  - `audit_log.py:9-19` `def log(...)` 第 14 行 `"input_hash": hash(str(inputs))[:16] if inputs else None`。
  - Python 语义：`hash(any_str)` → int；`int[:16]` → TypeError。可在线复现：`python -c "print(hash('x')[:16])"` → `TypeError: 'int' object is not subscriptable`。
  - 对照同目录 `utils/hash_input.py:8`：`hashlib.sha256(data.encode()).hexdigest()[:16]` —— **正确的实现就放在旁边**，`audit_log.py` 却用内置 `hash()` 重写且用错。
  - 附带：`hash()` 还受 `PYTHONHASHSEED` 影响，进程间不确定，即便不崩也不能作内容指纹。
- **remediation**: `import hashlib`；将该行改为 `"input_hash": hashlib.sha256(str(inputs).encode()).hexdigest()[:16] if inputs else None`（与 `hash_input.py` 对齐）。顺带把 `datetime.utcnow()`（line 11, 已 deprecated 返 naive）改为 `datetime.now(timezone.utc)` 与其他检测器一致。1 文件。

### GD-M-001 · medium · silent-failure/security-gate · route-fix
- **file**: detectors/number_hallucination.py
- **line**: 105-126（`check_reasonableness` 整个函数）；规则表 23-38
- **finding**: `check_reasonableness` 的 docstring 声称"检查数字是否在合理范围内"，加载 `FINANCE_RULES`（pe_ratio 0-500 / revenue_growth_pct -100..1000 / market_cap_B 0-10000 / percentage -100..100），但函数体**只 parse `val` 写回 `num["numeric_value"]` 后 `continue`，从不把 `val` 与规则 min/max 比对，也不产生任何 issue**。规则字典是死代码。"数字是否在合理范围"这一检查项是 **no-op**。
- **evidence**:
  - `number_hallucination.py:116-126` 循环体：`raw = re.sub(...); val = float(raw); num["numeric_value"] = val` 后 `continue`——无 `if val < rules[...]["min"]` 之类的判断，无 issues 列表追加。
  - `detect()`（141-174）的 `critical` 只来自"精确数字无 URL 来源"（152-160），**从不**因数值越界而 critical。
  - 数据流后果：`run_all_detectors.py` → `executor.phase4_complete`（executor.py:361-368）读 `overall=="PASS"` 作为 phase4 门禁。L1 越界检查 no-op → "PE = 99999"、"营收增长 5000%"、"市值 99999B" 这类 LLM 幻觉越界数字**静默通过门禁**。phase4 的 `must_pass` 门（state_manager.py:293-302）形同虚设。
- **remediation**: 在 `check_reasonableness` 中按数字上下文分类（`%` → percentage 规则；`$¥€£`+`BMK`/`亿/万` → market_cap_B；含 `PE`/`市盈` 邻域 → pe_ratio；etc），把 `val` 与所选规则的 min/max 比，越界则向该 num 追加 `{"range_issue": "...", "rule": ...}`；`detect()` 中把含 `range_issue` 的数字计入 `critical`（或新建 `range_issues` 数组并入 `passed` 判定）。1 文件。

### GD-M-002 · medium · correctness · route-fix
- **file**: scripts/computation/data/cross_check.py
- **line**: 18-21（`avg = (a+b)/2; diff_pct = (diff/abs(avg)*100) if avg else 0`）
- **finding**: 当两源值符号相反、绝对值相近时 `avg ≈ 0`，`if avg else 0` 走 falsy 分支 → `diff_pct = 0` → `consistent = (0 <= tolerance_pct) == True`。**符号相反的严重分歧被报为一致**。
- **evidence**: `cross_check.py:18-21`。例：`value_a=100, value_b=-100` → `diff=200, avg=0` → `diff_pct=0` → `consistent=True`。交叉验证本应catch 两源打架，反而 false-pass。
- **remediation**: 分母改用 `max(abs(a), abs(b), 1e-9)`（以两源中较大者为基准），或在 `avg==0 and diff>0` 时显式置 `consistent=False`。1 文件。

### GD-M-003 · medium · data-integrity · route-fix
- **file**: scripts/orchestration/state_manager.py
- **line**: 369-380（`save()` 写两个 state 文件）
- **finding**: `save()` 对 `state_file` 与 `analysis_dir/checkpoint.json` 都用 `json.dump` 直接写——**非原子**。崩溃发生于写入中途（或两文件之间）→ 文件截断 → `load()`（390-391）`json.load` 抛 JSONDecodeError → **断点续传失败**，而这正是 `ResearchState` 类存在的核心目的。同类已修 bug：x-tool OTA-M-003（monitor_state.py）/ OTA-L-004（storage.py）的 temp+os.replace 原子写。
- **evidence**: `state_manager.py:374-380` 两个 `with open(...,'w') as f: json.dump(...)`，无 temp file、无 `os.replace`。`load()`（390）直接 `json.load(open(state_file))`，截断即崩。
- **remediation**: 两个写都改为 `tmp = path + '.tmp'; json.dump(..., open(tmp,'w')); os.replace(tmp, path)`。1 文件。

### GD-L-001 · low · correctness · route-fix
- **file**: scripts/computation/finance/dcf.py
- **line**: 45-46（`value_per_share = enterprise_value / shares_outstanding`）
- **finding**: DCF 算出的是**企业价值（EV）**，每股**股权**价值应为 `(EV - net_debt) / shares`。此处直接 `EV/shares` 得到的是"每股企业价值"，却被标为 `value_per_share`。用户/Agent 若据此比价，会按净债务/股数高估公平价值。
- **evidence**: `dcf.py:36-46`：`enterprise_value = sum(pv_fcfs)+pv_terminal`；`result["value_per_share"] = round(enterprise_value / shares_outstanding, 2)`。无 net debt 减项。
- **remediation**: 要么改名 `enterprise_value_per_share`（如实），要么加 `--net_debt` 参数算 `(EV - net_debt)/shares`。1 文件。

### GD-L-002 · low · robustness · route-fix
- **file**: scripts/computation/finance/monte_carlo.py
- **line**: 37（`statistics.stdev(results)`）
- **finding**: `--simulations 1`（或随机导致 results 长度 1）时，`statistics.stdev` 需 ≥2 样本，抛 `statistics.StatisticsError` 未捕 → 脚本崩溃。默认 10000 不触发，但用户可显式传 1。
- **evidence**: `monte_carlo.py:14` `--simulations type=int default=10000`（无 `choices`/最小值校验）；`monte_carlo.py:37` `statistics.stdev(results)` 无 len 守卫。`descriptive.py:18` 同类已用 `if len(d)>1 else 0` 守卫。
- **remediation**: `stdev(...) if len(results) > 1 else 0`（对齐 descriptive.py 模式）。1 文件。

### GD-L-003 · low · robustness · route-fix
- **file**: scripts/computation/stats/timeseries.py
- **line**: 27-31（`cagr` 分支 `(d[-1]/d[0])**(1/years)`）
- **finding**: 当终值为负（亏损公司末年亏损）而初值正时，`d[-1]/d[0] < 0`，对负数取分数次幂 → Python 抛 `ValueError`/复数。CAGR 对负终值本就无定义，应返 None 而非崩。
- **evidence**: `timeseries.py:27-31` 只守卫 `len(d)>=2 and d[0]>0`，未守卫 `d[-1]>0`。`d[-1]<0` 时 `(-x)**(1/years)` 报错。
- **remediation**: 守卫加 `and d[-1] > 0`，否则 `result = None`。1 文件。
