- **路径遍历防护到位**：`storage.FileTaskStore.task_path`（storage.py:14-23）与 `artifacts.ArtifactStore.run_dir/path_for`（artifacts.py:15-41）均显式拒绝 `..`/`/`/`\` 并 `.relative_to()` 校验 resolve 后路径。良好。

- **Discord token 主路径已脱离 config**：`discord_bot.py:143-150` 显式只从 env 读 token，并注释"不再支持从 config/app.json 读取 secret"。

- **API 错误分层**：`api.py:15-28` 区分 client-error（400）与 server-error（500 不泄露异常文本）。

## Findings

### OTA-SEC-01 · critical · security · direction-shift
- **file**: interfaces/api.py
- **line**: 31-127（整个 FastAPI app 无任何鉴权依赖/中间件）；execute 端点 74-87 / 89-118
- **finding**: HTTP API 全部端点无鉴权，含 `/v1/tasks/execute` 与 `/v1/tasks/execute_structured`——二者直接驱动 Playwright CDP 操作已登录的美团后台浏览器，触发**真实金钱改价**。
- **evidence**: `create_api_app` 返回 `FastAPI(title="OTA Price Automation", version="0.1.0")` 后直接 `@app.post("/v1/tasks/execute")`，无 `Depends(...)` / 无 middleware。`execute` 调 `service.execute_text(..., auto_confirm=bool(payload.get("auto_confirm", False)))`。CLI 默认 host=127.0.0.1（cli.py:94），但 `--host` 或 `config/app.json` 的 `api.host` 可设 `0.0.0.0`（monitor.py:113 与 hermes_discord_bot.py:308 都显式处理 `0.0.0.0`/`::`，说明 0.0.0.0 是受支持的部署形态）。一旦绑 0.0.0.0，同网段任意主机可 POST 触发改价。
- **remediation**: 给所有 mutating 端点加鉴权依赖（最低：`X-API-Key` header 校验，key 从 env 读）；默认绑 127.0.0.1；`/execute*` 额外要求 admin 级 key。属公共接口契约变更 → direction-shift。

### OTA-SEC-02 · critical · security · direction-shift
- **file**: orchestrator.py / interfaces/api.py
- **line**: orchestrator.py:116-126（execute_text）、128-138（execute_structured）、124/136 `if auto_confirm or not record.risk.requires_confirmation: return self.confirm_task(...)`；api.py:80 `auto_confirm = bool(payload.get("auto_confirm", False))`
- **finding**: API 调用方可通过 `auto_confirm=True` 一键绕过 RiskGuard 的 `requires_confirmation`，配合 SEC-01 实现**无鉴权、无人确认、单次 POST 即改价**。
- **evidence**: `RiskGuard.evaluate`（risk.py:11-31）默认 `require_confirmation=True`，但 `execute_text` 在 `auto_confirm=True` 时直接 `confirm_task`，跳过 waiting_confirmation 状态。无任何 source 区分：API 与 CLI 共用同一路径，API 也能传 `auto_confirm=True`。
- **remediation**: 当 `source=="api"` 时拒绝 `auto_confirm=True`，或要求额外 admin 凭证；至少对 `source!="cli"` 强制走 preview→confirm 两步。direction-shift（改变执行契约）。

### OTA-AUDIT-01 · high · audit-log-gap · route-fix
- **file**: platforms/meituan.py / orchestrator.py / interfaces/api.py
- **line**: meituan.py:30-134（change_price 全程）、orchestrator.py:152-170（confirm_task）、api.py:74-118（execute 端点）
- **finding**: 整条改价执行链路**无 audit-log 埋点**。改价是金钱性外部动作（CDP 浏览器自动化 + 美团后台写操作），命中 autonomous-constraints.md B 节"外部调用 / 变更 / 部署"敏感路径，必须按 `.claude/decisions/audit-log.schema.json` 落 JSONL。
- **evidence**: 模块内无 `audit_log.py`/`.audit/` 目录。改价详情只写 `artifacts/.../execution-details.json`（meituan.py:79-85）与 `data/tasks/<id>.json`（storage.py:25-27），均非 schema 化审计日志（缺 `id`/`timestamp`/`userId`/`action`/`resource`/`result`/`ip` 字段）。`confirm_task` 入口/出口、`change_price` 入口/出口、API execute 端点均无审计事件。失败路径（meituan.py:105-134）只记 `str(exc)` 到 details，不产审计 `result=failure` 事件。
- **remediation**: 新增最小 `ota_workflow/audit_log.py`（append-only JSONL 写 `<data_dir>/.audit/audit-YYYY-MM-DD.jsonl`，字段对齐 schema）；在 `confirm_task` 入口/出口埋 `action=update, resource.type=ota_price, resource.identifier=task_id, userId=parsed.metadata.discord_user_id or source, ip=local`，`result` 如实反映 success/failure。1 新文件 + 2-3 处调用 → route-fix。

### OTA-SEC-03 · high · secrets-hygiene · route-fix
- **file**: monitor.py / interfaces/hermes_discord_bot.py
- **line**: monitor.py:128 `token = os.getenv(token_env_var) or discord_config.get("token")`；hermes_discord_bot.py:305 `token = os.getenv(token_env_var) or discord_config.get("token")`
- **finding**: Discord token 仍保留"从 config/app.json 读取"的回退路径，与 `discord_bot.py` 已显式删除的回退不一致；若 token 被提交进 `config/app.json` 将随 git 泄露且无告警。
- **evidence**: `discord_bot.py:145-150` 已改为 env-only 并注释"不再支持从 config/app.json 读取 secret"。但 `monitor.py` 的 `DiscordNotifier.from_settings` 与 `hermes_discord_bot.py` 的 `run_bot` 仍 `or discord_config.get("token")`。`hermes_discord_bot.py:306` 只挡了字面量占位符 `"PUT_DISCORD_BOT_TOKEN_HERE"`，挡不住真实 token 被写进 config。
- **remediation**: 删除两处 `or discord_config.get("token")` 回退，统一 env-only；缺失即 raise。route-fix（2 文件各 1 行）。

### OTA-SEC-04 · medium · security · route-fix
- **file**: interfaces/hermes_discord_bot.py
- **line**: 176-184（`_build_discord_connector`），182 `if hasattr(ssl, "OP_NO_TLSv1_3"): ssl_context.options |= ssl.OP_NO_TLSv1_3`
- **finding**: 配置代理时显式禁用 TLS 1.3，把连接降级到 TLS 1.2 上限，无注释说明原因。
- **evidence**: `_build_discord_connector` 在 `proxy_url` 存在时构造 `ssl.create_default_context()` 后 `options |= ssl.OP_NO_TLSv1_3`，再 `aiohttp.TCPConnector(ssl=ssl_context)`。`create_default_context()` 默认已协商最高双方支持版本，无需手动禁 1.3；该行削弱前向安全。
- **remediation**: 删除 `OP_NO_TLSv1_3` 行；若确有代理兼容问题，加注释 + 限定仅对失败握手降级。route-fix（1 行）。

### OTA-CORR-01 · medium · reliability · route-fix
- **file**: monitor.py
- **line**: 81-88（`run_forever`）
- **finding**: 巡检主循环 `except Exception` 全吞，仅 `print`，无退避、无持续失败告警——状态文件损坏等持久故障会每 30s 日志轰炸且无 Discord 告警。
- **evidence**: `run_forever` 内 `try: report = self.run_once() ... except Exception as exc: self._log(f"巡检失败: {exc}")` 后 `time.sleep(max(self.config.interval_seconds, 30))`。无连续失败计数、无指数退避、不调 `_send_alert`。
- **remediation**: 引入连续失败计数器，超过阈值（如 3 次）发一次 Discord 告警并指数退避。route-fix（单文件）。

### OTA-CORR-02 · medium · security · direction-shift
- **file**: monitor.py / interfaces/hermes_discord_bot.py
- **line**: monitor.py:200-243（review_failure，230 把 review_prompt 含 metadata json.dumps 给 LLM）、318-341（`_build_failure_payload` 第 340 行 `"metadata": dict(record.parsed.metadata or {})`）；hermes_discord_bot.py:256-291（chat，285-288 把 LLM `content` 原样回贴 Discord）
- **finding**: 失败回查把含 PII（discord_user_id / channel_id）的 `metadata` 整包喂给 Hermes LLM；hermes-discord bridge 把 LLM 自由文本输出**原样回贴 Discord**——prompt-injection / 数据外渗 / 伪造 task_id 风险。
- **evidence**: `_build_failure_payload` 返回 `metadata` 字段，`review_failure` 把整个 `failure_payload` 作为 `task` 塞进 `review_prompt` 发给 LLM（monitor.py:204-211, 230）。`HermesApiClient.chat` 把 `message.content` 拼进 user payload 发 LLM，返回 `content` 直接 `_split_content` 后回贴 Discord（hermes_discord_bot.py:285-294）。`SYSTEM_PROMPT` 仅"口头"要求"Do not invent task ids"，无强制输出校验。
- **remediation**: 失败回查 payload 去除 PII 字段；bridge 侧对 LLM 输出做结构化校验（task_id 必须匹配 `TASK_ID_PATTERN` 且为已存在任务），优先用 tool-calling 而非自由文本。属交互契约变更 → direction-shift。

### OTA-CORR-03 · low · reliability · route-fix
- **file**: monitor_state.py
- **line**: 39-40（`_save_json` → `path.write_text(...)`）
- **finding**: state/routes/last_run JSON 非原子写；崩溃中途写坏 → `load_*` 静默返回 default，丢失告警去重 `events` → 重复告警风暴或漏告警。
- **evidence**: `_save_json` 直接 `path.write_text(json.dumps(...))`，无 temp+rename。`_load_json` 捕获 `JSONDecodeError` 返回 default（monitor_state.py:32-34），半写文件被静默重置。
- **remediation**: 写临时文件后 `os.replace` 原子替换。route-fix（单文件）。

### OTA-CORR-04 · low · data-integrity · route-fix
- **file**: storage.py
- **line**: 25-27（`save`）
- **finding**: 任务记录非原子写；崩溃中途写坏丢整条 task 记录（含改价结果）。
- **evidence**: `path.write_text(json.dumps(to_jsonable(record), ...))` 直接写。
- **remediation**: temp + `os.replace`。route-fix（单文件）。

### OTA-CORR-05 · low · correctness · route-fix
- **file**: parser.py
- **line**: 89-105（`_resolve_room_type`，96 `elif room_fragment in alias or alias in room_fragment`）
- **finding**: 房型模糊匹配可能误命中——子串命中且唯一时直接采纳，可能改错房型价格。
- **evidence**: 模糊匹配 `room_fragment in alias or alias in room_fragment`，唯一命中即返回（99-100 `matches = exact_matches or fuzzy_matches`）。例如"大床"子串命中"大床房"。多命中才 raise（103-104）。
- **remediation**: 模糊命中需长度比阈值或强制二次确认。route-fix（单文件）。
