### AE-NEW-001 [MEDIUM] save-checkpoint.py write_checkpoint 非原子写 — 崩溃损坏恢复文件
- **文件**: `.claude/hooks/save-checkpoint.py`
- **行号**: 187-192
- **类型**: route-fix
- **finding**: `write_checkpoint()` L187 `with open(filepath, "w") as f: json.dump(...)` 先截断再写，无 tempfile/fsync/os.replace；L192 `shutil.copy2(filepath, LATEST_LINK)` 也是非原子覆盖。若进程在 `json.dump` 中途崩溃（SSH 断开 / OOM / kill），`filepath` 与 `latest.json` 均为截断残缺 JSON。下次 SessionStart `resume-checkpoint.py` `_read_json()` 解析失败返回 None → 整个检查点恢复链路断裂——**而检查点存在的全部意义就是崩溃恢复**。同项目 `incremental-save.py` L42-58 已实现正确的 `tempfile.mkstemp` + `tmp_f.flush()` + `os.fsync()` + `os.replace` 原子写模式，可直接移植。
- **remediation**: 移植 incremental-save.py 的原子写模式到 `write_checkpoint()`：tempfile 在同目录 → fsync → os.replace 覆盖 filepath；再 os.replace(filepath, LATEST_LINK)（或对 latest.json 也走 tempfile+replace）。
- **证据**: L187 `with open(filepath, "w", encoding="utf-8") as f: json.dump(...)`；L192 `shutil.copy2(filepath, LATEST_LINK)`。对比 incremental-save.py L45-51 正确模式。
- **分类**: 数据损坏 / 崩溃鲁棒性 race（反 churn 例外条款"race/数据损坏"明确允许）。

### AE-NEW-002 [LOW] watchdog.sh stat 失败致 stale marker 静默删除 — 丢失中断信号
- **文件**: `.claude/watchdog.sh`
- **行号**: 86-89
- **类型**: route-fix
- **finding**: L86 `MARKER_AGE=$(($(date +%s) - $(stat -c %Y "$STALE_MARKER" 2>/dev/null || echo 0)))`。若 `stat` 失败（TOCTOU：STALE_MARKER 在 L85 `[ -f ]` 检查与 L86 stat 之间被删；或权限问题），`|| echo 0` 使减数变 0 → `MARKER_AGE = date +%s ≈ 1.7e9` → L87 `[ $MARKER_AGE -gt 1800 ]` 恒真 → L88 `rm -f "$STALE_MARKER" "$RESUME_FLAG"` 静默删除有效标记。后果：上次会话异常中断的信号被吞掉，SessionStart 不再报告中断历史。
- **remediation**: stat 失败时 `continue`/跳过删除而非按 0 算年龄：`stat_ts=$(stat -c %Y "$STALE_MARKER" 2>/dev/null)` + `if [ -z "$stat_ts" ]; then continue; fi`。
- **证据**: L86 `$(stat -c %Y "$STALE_MARKER" 2>/dev/null || echo 0)` + L87 `if [ $MARKER_AGE -gt 1800 ]` + L88 `rm -f`。
- **分类**: 静默吞错（反 churn 例外条款明确允许）。

### AE-NEW-003 [LOW] notify-phone.py finally sock.close() 在 socket.socket() 失败时 UnboundLocalError — 破坏静默契约
- **文件**: `.claude/hooks/notify-phone.py`
- **行号**: 69-78
- **类型**: route-fix
- **finding**: `notify_via_tunnel()` L70 `sock = socket.socket(...)` 若抛 OSError（EMFILE/ENFILE 文件符耗尽，hook 高频触发时可能），`sock` 未绑定 → `except (ConnectionRefusedError, socket.timeout, OSError): return False` 捕获 → 但 `finally` L78 `sock.close()` 先于 return 执行 → `UnboundLocalError: local variable 'sock' referenced before assignment` → 该异常未被捕获，从 `send()`(L175 调用未 try) 与 `main()`(L206 未 try) 一路抛出 → hook 进程崩溃、traceback 喷 stderr → 违反本 hook 设计原则 L9"所有错误静默处理，不影响 Claude Code 正常运行"。
- **remediation**: L70 前置 `sock = None`；finally 改 `if sock is not None: sock.close()`。或把 socket.socket 移出 try。
- **证据**: L69-78 `try: sock = socket.socket(...)... except (...,OSError): return False finally: sock.close()`——sock 在 socket.socket 抛异常时未绑定。
- **分类**: 异常处理失效 / 断言失效（反 churn 例外条款"断言失效/静默吞错"允许）。
