## Scope

验证 `audit-cycle-state.json` 中 `derived_fixes[].status=merged` 的声明是否对应 main 分支上**实际存在的代码变更**。聚焦最近一轮 cycle（audit-038）及前序 meta-audit fix（audit-032/033/036 派生）。

## Methodology

1. `git worktree list` 枚举所有活跃 worktree
2. 对每个 worktree HEAD commit，检查 `git branch --contains <sha>` 是否在 main
3. 不在 main 的 commit，通过 `git log main --grep=<finding_id>` 查找 squash-merge 等价
4. 对声称 merged 的 fix，验证其**关键文件改动**是否出现在 main 历史中（`git show --name-only` + `git grep`）
5. 交叉比对 structural-debt.md on main vs audit-cycle-state history 引用的 SD ID 集合
