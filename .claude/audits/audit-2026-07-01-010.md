## H-001: 文档全面假定 Windows 环境，Linux 部署不可操作
- **file**: CLAUDE.md, local_simulator/README.md, local_simulator/USAGE.md
- **line**: CLAUDE.md:37-48, README.md:72-250, USAGE.md:7-343
- **severity**: HIGH
- **finding**: 所有项目文档（CLAUDE.md 快速命令、README 完整部署指南、USAGE 操作手册）均使用 Windows 路径（`E:\x-tool\moni`, `F:\stock-backtest\data\stocks_minute.duckdb`）和 PowerShell 命令（`Set-Location`, `powershell -ExecutionPolicy Bypass`）。Linux 部署（当前实际环境 `/home/admin/workspace/moni-master`）无法按文档执行任何操作。
- **remediation**: 将文档中的绝对路径改为相对路径或环境变量引用；PowerShell 命令补充 bash/zsh 等价命令；添加"跨平台部署"章节说明 env var override（MONI_QLIB_PROVIDER_URI 等）。
- **kind**: route-fix（纯文档改动，不影响接口）

## H-002: factors/ 脚本 70+ 处 Windows 路径硬编码（结构性技术债）
- **file**: factors/*.py, factors/scripts/**/*.py, factors/.hermes/skills/*/scripts/*.py
- **line**: 见 grep 结果（multi_factor_round2.py:7-8, bt_fast.py:9-10, alphagraph_t0_backtest.py:16-17, cb_backtest/config.py:8-9, alpha_continuous_ctl.py:14,23, validate_accepted_factors.py:35,170 等 70+ 处）
- **severity**: HIGH
- **finding**: 因子研发层大量脚本在模块顶层硬编码 `F:/stock-backtest/data/stocks_minute.duckdb`、`E:/x-tool/moni/...`、`E:\\openclaw\\...` 等 Windows 路径。与 local_simulator 核心的 config+env 健康设计形成鲜明对比。Rust 重写（moni-core/paths.rs）已解决此问题，但 Python 旧代码仍携带债务。**这是结构性问题**：需要统一的路径解析模块（类似 paths.rs 的 Python 版）供所有 factors 脚本导入。

## M-001: engine.rs attempt_match 错误被静默吞掉
- **file**: moni-rs/crates/moni-engine/src/engine.rs
- **line**: 467
- **severity**: MEDIUM
- **finding**: `update_quotes` 中驱动 open orders 撮合时：`let _ = self.attempt_match(&mut order, &q, "quote_update");`。`attempt_match` 返回 `Result<(), String>`，错误被 `let _ =` 完全丢弃。后果：(1) 撮合异常不入日志 (2) 客户端收不到 error event (3) 订单卡在 SUBMITTED/PARTIALLY_FILLED 状态无解释。Python 原版有 logging.warning。
- **remediation**: 改为 `if let Err(e) = self.attempt_match(...) { tracing::warn!(order_id=%order.order_id, err=%e, "match failed on quote update"); self.notifications.publish(EngineEvent::new("match_error", ...)); }`
- **kind**: route-fix

## M-002: memory.rs RwLock unwrap 无 poison 恢复
- **file**: moni-rs/crates/moni-data/src/memory.rs
- **line**: 28, 45, 53, 63, 71
- **severity**: MEDIUM
- **finding**: 所有 `self.quotes.read().unwrap()` / `.write().unwrap()` 在 RwLock poisoned 时会 panic。当前单 actor 模型下 poison 概率极低（仅当 holder panic 时发生），但：(1) 未来若引入多线程 writer 风险上升 (2) 违反 Rust 防御性编程惯例 (3) 数据库/网络服务通常用 `unwrap_or_else(|e| e.into_inner())` 降级。
- **remediation**: 将所有 `.unwrap()` 改为 `.unwrap_or_else(|e| { tracing::error!("rwlock poisoned, recovering"); e.into_inner() })`。或在 type alias 层面封装 `PoisonSafeRwLock<T>`。
- **kind**: structural（跨多处统一模式，建议封装 helper）

## M-003: config.rs canonicalize 静默 fallback 掩盖缺失配置
- **file**: moni-rs/crates/moni-core/src/config.rs
- **line**: 339
- **severity**: MEDIUM
- **finding**: `path.canonicalize().unwrap_or_else(|_| path.to_path_buf())` — 当 config.yaml 不存在或路径无效时，静默回退到原始（可能是相对）路径，随后 `read_to_string` 才报 Read error。但 base_dir 计算（line 340）基于这个可能无效的 abs，导致 storage 路径解析错误。更严重的是：如果传入一个存在的目录而非文件，canonicalize 成功但 read_to_string 给出 confusing "Is a directory" 错误。
- **remediation**: 移除 unwrap_or_else，让 canonicalize 错误向上传播：`let abs = path.canonicalize().map_err(|e| ConfigError::Read(path.display().to_string(), e))?;` 并在 read_to_string 前加 `if !abs.is_file() { return Err(ConfigError::Read(..., io::Error::new(InvalidInput, "not a file"))); }`
- **kind**: route-fix

## M-004: engine.rs 恒等 map_err 死代码
- **file**: moni-rs/crates/moni-engine/src/engine.rs
- **line**: 298
- **severity**: MEDIUM (代码卫生)
- **finding**: `let result = self.place_order(request, &source); let _ = reply.send(result.map_err(|e| e));` — `map_err(|e| e)` 是恒等变换，无实际作用。可能是重构残留（原计划转换错误类型但未完成）。增加认知负担，clippy 应能捕获。
- **remediation**: 删除 `.map_err(|e| e)`，直接 `reply.send(result)`。运行 `cargo clippy -- -W clippy::identity_conversion` 确认无其他同类。
- **kind**: route-fix

## Cross-cutting Observations

### ✅ 健康设计（值得肯定）
1. **local_simulator 核心运行时**：完全通过 config.yaml + MONI_* env var 解析路径，零硬编码。models.py:249 `os.environ.get("MONI_QLIB_PROVIDER_URI") or config[...]` 是正确模式。
2. **moni-core/paths.rs**：env-first + 相对默认 + 完整测试覆盖。Rust 重写已系统性解决路径问题。
3. **config.rs**：serde default + env override + storage 路径解析 + 测试（含 EnvGuard RAII 防并行污染）。高质量。
4. **engine.rs actor model**：mpsc command channel + oneshot reply + broadcast events。清晰的并发边界。测试覆盖买卖往返、撤单、风控拒绝、事件广播。

### ⚠️ Python ↔ Rust 双轨期的断裂面
- Rust crates 已实现正确的路径抽象，但 Python factors/ 仍是旧模式。两套代码共存期间，同一数据源的路径解析逻辑不一致。建议加速 Rust 迁移或在 Python 侧 backport paths.rs 语义。
