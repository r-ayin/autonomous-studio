### PA-ASR-01 — HIGH — asr.py 死代码回归：scan_once/run_loop 仍嵌在 query_habits 内（prior fix case-2026-07-02-131 伪成功）

- **file**: `personal_assistant/asr.py`
- **line**: 186-202（`scan_once` 186-197、`run_loop` 199-202，缩进在 `query_habits` 函数体内、line 184 `return` 之后）
- **kind**: route-fix
- **evidence**:
  - `IngestionPipeline` 类体止于 line 159（`_analytics` 方法结束）。`query_habits()` 是模块级函数（162-184），line 184 `return {"daily":..., "total_segments": meta[1]}` 后即结束。
  - line 186 `def scan_once(self)` 与 line 199 `def run_loop(self, poll_seconds=10.0)` 缩进 4 空格——落在 `query_habits` 函数作用域内、且在 return 之后 → **永不执行**。两者均用 `self`、调 `self.process_file`，明显原属 `IngestionPipeline`。
  - 实测：`IngestionPipeline().scan_once()` → `AttributeError: 'IngestionPipeline' object has no attribute 'scan_once'`。类 docstring（line 108 "轮询 inbox"）承诺的能力不存在。
  - **回归历史**：07-01-006 PA-H-003（HIGH）与 07-02-008 M-003（medium）**两次**报此 bug；case-2026-07-02-131 work_unit="asr.py dead code relocation (M-002)" 声称 `outcome=succeeded`，但 `outcome_evidence` 仅 `python3 -c ast.parse OK + opt-worktree.sh commit succeeded + branch pushed`——**没有任何证据证明死代码被移走**。`git log -- asr.py` 显示后续只 e6c49ce（analytics refactor）/ 3762df9（time_kind）动过此文件，均与死代码无关。死代码原封不动。

### PA-WIKI-01 — MEDIUM — wiki.py `_body_grounded` 单 bigram 命中即过，反幻觉形同虚设

- **file**: `personal_assistant/wiki.py`
- **line**: 27-35（`_body_grounded`），调用点 86-89
- **kind**: route-fix
- **evidence**: `_body_grounded` 遍历 body 的 2 字 bigram，**首个**命中 src_text 即 `return True`（line 33-34）。中文常用 bigram（"的"/"是"/"不"/"在"+任一字）几乎必然出现在任意源文本里 → 一篇几乎全幻觉的 body 只要含任一通用 bigram 即过 grounding。docstring（line 4 "body bigram 落地全部 source_ids 内容"）与 inline 注释（line 86 "body 须落地全部最终 source_ids 内容"）都声称验证 body 落地**全部**源，实现只要求**任一** bigram。声明与实现矛盾。
- **impact**: wiki 页 body 可大段幻觉而通过反幻觉门，污染个人知识库。与 07-05-002 PA-PII-01 报的 `verify.check_memories` bigram 弱点（verify.py:159）**同族但不同函数**——verify 那条已入 SD，wiki 这条此前无人报。
- **remediation**: 改为覆盖率阈值——统计 body bigram 命中 src_text 的比例，要求 ≥50%（或 ≥min(N, len(bigrams)//2) 个不重叠命中）方算落地。1 文件 route-fix。

### PA-REC-01 — MEDIUM — recommend.py grounding 在搜索结果标题为空时放行任意 LLM 幻觉 item

- **file**: `personal_assistant/recommend.py`
- **line**: 62, 94-95（`grounded`）
- **kind**: route-fix
- **evidence**: `res_titles = [_norm(r.get("title","")) for r in (results or [])]`（line 62）。若搜索结果 `title` 全空（如 Baidu/某 scraper 返 `{"title":""}`），`_norm("")` = `""`，`res_titles = ["","",...]`（**truthy 非空 list**）。`item_ok = (not res_titles) or any(item and (item in t or t in item or _share_token(item,t)) for t in res_titles)`（line 94）——短路 `not res_titles`=False 故走 `any(...)`；对每个空 `t`，`t in item`（即 `"" in "任意字符串"`）在 Python 恒为 True（空串是所有串子串）→ **任意 item 落地**，反幻觉门洞开。
- **impact**: 当搜索源返空标题（scraper 故障/反爬），LLM 编造的任意 item 都会作为"真实推荐"输出，违背模块核心反幻觉承诺（docstring line 3 "不写死任何推荐内容，不编造"）。
- **remediation**: `res_titles = [t for t in (_norm(r.get("title","")) for r in (results or [])) if t]`——过滤掉空规范标题，使 `not res_titles` 分支在"无可用标题"时正确触发。1 文件 route-fix。
