## Findings

### H-001 Strategy sandbox escape via attribute-chain bypass (HIGH)

- **file**: factors/scripts/qlib_repo/trader/frontend/api_server.py
- **line**: 295-318 (validate_strategy_source), 3588-3592 (save after validate), backtest_engine.py:432 (exec)
- **severity**: high
- **kind**: route-fix
- **finding**: validate_strategy_source 用 AST walk 检测危险标识符，但仅检查 `ast.Name.id` 和 `ast.Attribute` 链的字符串包含。攻击者可用以下 payload 绕过：
  ```python
  # 绕过 __import__ 黑名单
  g = globals()
  b = g.get('__builtins__') if isinstance(g.get('__builtins__'), dict) else g['__builtins__'].__dict__
  b['open']('/tmp/pwn','w').write('x')
  # 或通过 object.__subclasses__
  ().__class__.__bases__[0].__subclasses__()
  ```
  restricted_import (backtest_engine.py:380-398) 仅拦截顶层 import name，未阻断 getattr + 字符串动态分发。validate 通过后 save_strategy_source 落盘，后续 execute_backtest_request 调用 compile_strategy → exec(source)。
- **remediation**:
  1. 切换为白名单 AST visitor：仅允许 ast.Constant/ast.BinOp/ast.Compare/ast.Subscript/ast.Call(限定 func)/ast.Attribute(限定 attr chain) 节点类型，其他一律拒绝。
  2. exec 时使用 RestrictedPython 或 asteval 库做运行时 enforcement，而非仅静态预检。
  3. 在 namespace 里彻底移除 __builtins__（设为空 dict 而非受限副本），需要 print/len 等时显式注入安全包装。
  4. 增加 integration test：提交上述绕过 payload 应被拒。

### H-002 Factor expression eval with partially-open builtins (HIGH)

- **file**: factors/scripts/fast_t0_backtest.py
- **line**: 169, 136, 238
- **severity**: high
- **kind**: route-fix
- **finding**: `eval(fexpr, {'__builtins__': {'mul': lambda a,b: a*b}}, env)` 中 fexpr 来自模块级 FACTORS dict（当前硬编码，但若未来改为从 YAML/DB/UI 加载即成 RCE）。env 由 make_operator_env(df5) 构造，含 pandas Series/DataFrame 对象，其 `.values.__class__.__bases__[0].__subclasses__()` 可枚举所有 Python 类找到 os._wrap_close/subprocess.Popen。即使 __builtins__ 只有 mul，通过属性链仍可逃逸。同模式在 alpha_graph/strategy/*.py 重复出现 ≥8 处。
- **remediation**:
  1. 把 eval 替换为 pyparsing/lark 解析自定义 DSL，或 pandas.eval(engine='python') 配合 resolvers。
  2. 若保留 eval，至少把 __builtins__ 设为 `{}`（空 dict，非带 mul 的 dict），mul 通过 env 注入。
  3. 在 env 里只暴露 numpy 函数，不暴露 raw Series/DataFrame 对象（传 .values ndarray 代替）。
  4. 加单元测试断言 FACTORS 表达式不含双下划线属性访问。

### M-001 DuckDB f-string SQL injection in upsert_frame (MEDIUM)

- **file**: factors/scripts/sync_additional_local_feeds.py
- **line**: 242, 255 (upsert_frame); 同类模式 jqdata_minute_sync.py:603,617 / build_market_lake_hot_db.py:110 / setup_combined_patch_universes.py:254 / cb_minute_sync_sina.py:86 / setup_minute_universe_views.py:102 / setup_patch_aware_universes.py:98 / sync_free_market_metadata.py:241 / factor_mining/build_mad_intraday_factor_library.py:315,375,398
- **severity**: medium
- **kind**: route-fix
- **finding**: `con.execute(f"INSERT INTO {table_name} ({cols}) SELECT {cols} FROM {stage_table}")` 中 table_name/cols 来自函数参数或 DataFrame.columns。虽然 DuckDB 默认本地无网络攻击面，但若上游 CSV 含恶意列名（如 `"code; DROP TABLE stock_daily--"`）或 config.yaml 表名被篡改，则触发 SQL 注入。DuckDB 不支持 parameterized identifier，需显式 quote。
- **remediation**:
  1. 封装 `quote_identifier(name)` 工具函数：`return '"' + name.replace('"', '""') + '"'`，所有 f-string 表名/列名走此函数。
  2. 在 upsert_frame 入口加白名单校验：`re.fullmatch(r'[A-Za-z_][A-Za-z0-9_]*', table_name)`，不符则 raise ValueError。
  3. 集中到单一 db_utils.py 避免散弹修改。

### M-002 PostClosePipeline.ensure_service zombie leak on timeout (MEDIUM)

- **file**: local_simulator/post_close_pipeline.py
- **line**: 161-184
- **severity**: medium
- **kind**: route-fix
- **finding**: subprocess.Popen 启动子进程后进入 health check loop。若 deadline 超时 raise RuntimeError，但：
  1. 未调用 process.terminate()/kill()，子进程继续运行成为孤儿。
  2. 未设置 stdin=subprocess.DEVNULL，子进程继承父 stdin（交互式 shell 下会抢输入）。
  3. stdout/stderr 文件句柄 open("a") 后传给 Popen，异常路径未关闭 → fd 泄漏。
  4. 成功路径也未 wait/reap，依赖 daemon=True 行为不可靠。
- **remediation**:
  1. 用 try/finally 包裹：finally 里若未 returned success 则 process.kill() + process.wait(timeout=5)。
  2. 添加 stdin=subprocess.DEVNULL。
  3. 改用 contextlib.ExitStack 管理 stdout/stderr 文件句柄。
  4. 成功后保存 pid 到 pipeline_dir/service.pid 以便后续管理。

- **file**: factors/scripts/qlib_repo/trader/frontend/api_server.py
- **line**: 302-318
- **severity**: low
- **kind**: route-fix
- **finding**: `if pattern in chain_str` 对 "eval" 会匹配 "evaluate"、"retrieval" 等合法标识符（误报）；同时遗漏 `importlib.import_module`、`bytes.decode`、`type.__new__`、`object.__init_subclass__`、`code.InteractiveInterpreter` 等沙箱逃逸原语。AST-based blacklist 本质脆弱。

### I-001 Hardcoded Windows path in final_optimization.py (INFO)
