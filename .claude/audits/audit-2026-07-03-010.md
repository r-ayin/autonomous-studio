## 核心风险主题

### 1. Extracted Skill 与 Live 部署严重脱节（MBP-F009, 影响全部三模块）

这是本次审计最重要的结构性发现。主分支保护的 extracted 版本（194L）缺少 live 版本（517L）的所有加固：shlex tokenization、indirect unwrapping、command-chain segmentation、command-substitution detection、expanded dangerous flags、audit logging。**如果将 extracted skill 部署到新环境，所有已知 bypass 都会复活**。自动提交和编辑后检查也存在类似但程度较轻的版本漂移。

**建议**: 建立 CI check 或 pre-commit hook 确保 extracted skills 与 live 版本保持同步；或将 extracted skills 改为 symlink/reference 而非 copy。

### 2. 凭证泄露链（AC-F002 + AC-F005 + PEL-002 + PEL-010）

auto-commit 的 `git add -A` + 不完整排除列表 = 敏感文件可能被提交。post-edit-lint 的编译器错误注入 + dotfile 误判 = `.env.py` 内容可能泄露到 LLM context。两者形成端到端泄露链。

### 3. Gate Bypass 面（MBP-F001~F005）

extracted 版的 commit gate 有至少 5 种独立 bypass 路径。Live 版已修复但 extracted 版未同步。这不影响当前运行中的引擎（用 live 版），但影响 skill 的可移植性和可信度。

### 4. 零 Observability（AC-F008, PEL-008, MBP-F014）

三个模块都没有 audit-log 埋点，违反 autonomous-constraints.md B 节。无法事后追溯：多少次自动提交成功/失败？gate 拦了多少次？post-edit-lint 实际跑了多少次、命中率多少？
