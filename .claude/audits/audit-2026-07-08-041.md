## 项目概况

wechat-main 未审模块分两类：6 个目录（benchmark/, hotspot-entry/, persona/, planning/, prompts/, reference/）为纯文档/markdown，零可执行代码；tests/ 结构良好，用 tempfile+mock 隔离 I/O。真实 finding 集中在这些测试所覆盖的源文件中：CLI slug 参数未经 sanitize 直接流入路径构造和 shell 命令，形成 path traversal + shell injection 向量。persona 模板存在架构级间接 prompt injection 风险（web 搜索结果流入 LLM prompt 与可信 persona spec 并列），但属设计层面问题非代码 bug。

## HIGH Findings (1)

### WC-EXT-H-002 feedback_collector.py:62 — slug JSONL log injection
- **file**: scripts/feedback_collector.py:62-65, 136
- **severity**: high
- **finding**: `_append_reward(args.slug, ...)` writes user-controlled slug directly into JSONL record without sanitization. A slug containing newlines (`\n`) or crafted JSON fragments corrupts the JSONL format, enabling log injection. The slug comes from CLI arg `p_ingest.add_argument("slug")` at line 224 with no validation.
- **evidence**: Line 65: `"slug": slug,` embedded in dict then `json.dumps()` per-line. While json.dumps escapes control chars in values, a slug with literal newlines before JSON encoding would break line-oriented JSONL parsing. More critically, downstream consumers (line 172-191) use `rec.get("slug")` as sort key and display value without sanitization.
- **remediation**: Validate slug at CLI ingestion point against `re.fullmatch(r'[a-z0-9][a-z0-9_-]{0,63}', slug)`. Reject on mismatch. Apply same validation to all slug entry points.
- **kind**: route-fix

## MEDIUM Findings (3)

### WC-EXT-M-001 steps.py:256 — shell injection via slug in bash command strings
- **file**: scripts/steps.py:256
- **severity**: medium
- **finding**: `cmd=f"python scripts/competitor_analyzer.py {_research(slug, date, 'competitor-style')} > output/state/{slug}_competitor_metrics.json"` interpolates slug into shell commands. When slug arrives via `--slug` CLI override (pipeline.py:125), it bypasses `_slugify()` sanitization entirely.
- **evidence**: pipeline.py line 125: `slug = args.slug or _slugify(topic)` — raw args.slug used verbatim. Steps.build_steps() receives this unsanitized slug and embeds it in multiple shell command strings.
- **remediation**: Apply `_slugify()` to `args.slug` as well: `slug = _slugify(args.slug) if args.slug else _slugify(topic)`. Or validate against strict regex. Additionally, use `shlex.quote()` for all interpolated values in shell commands.
- **kind**: route-fix

### WC-EXT-M-002 pipeline.py:125 — --slug CLI override bypasses _slugify
- **file**: scripts/pipeline.py:125
- **severity**: medium
- **finding**: Root cause of WC-EXT-M-001. When `--slug` is provided, it is used verbatim without any validation. This raw slug flows into `_state_path()`, `_run_gate()` shell commands, `steps.build_steps()` shell commands, and file paths throughout the pipeline.
- **evidence**: `slug = args.slug or _slugify(topic)` — conditional sanitization means explicit user input is LESS safe than auto-generated input.
- **remediation**: `slug = _slugify(args.slug) if args.slug else _slugify(topic)`. Single line fix, eliminates entire class of downstream vulnerabilities.
- **kind**: route-fix

### WC-EXT-M-003 knowledge_base.py:75 — unbounded file read
- **file**: scripts/knowledge_base.py:75
- **severity**: medium
- **finding**: `text = article_path.read_text(encoding="utf-8")` reads entire article file into memory without size limit. A multi-gigabyte file passed via `--article` exhausts memory.
- **evidence**: No `stat().st_size` check before `read_text()`.
- **remediation**: Add size guard: `if article_path.stat().st_size > 10 * 1024 * 1024: raise ValueError("article too large")`. Same pattern needed in persona_drift.py:90.
- **kind**: route-fix

## LOW Findings (2)

### WC-EXT-L-001 research_cache.py:166 — non-atomic cache write
- **file**: scripts/research_cache.py:166
- **severity**: low
- **finding**: `cache_path.write_text(json.dumps(...))` direct overwrite without write-to-temp+rename. Crash mid-write leaves corrupted JSON. Graceful degradation exists (cmd_get reports cached=False), but data silently lost.
- **remediation**: Write to `cache_path.tmp` then `os.rename()` for atomic replacement.
- **kind**: route-fix
