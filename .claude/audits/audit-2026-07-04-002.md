## Findings

### HIGH

#### MM-H001 — settle_buy_fill can drive frozen_cash negative
- **File**: `moni-rs/crates/moni-engine/src/account.rs` L87-93
- **Kind**: route-fix
- **Finding**: When `actual_cost + new_frozen > previous_frozen` (possible with rounding drift across multiple partial fill settlements), `frozen_cash` goes negative because the subtraction is unconditional while the refund is clamped to `max(0)`.
- **Remediation**: Add floor clamp after adjustment: `self.state.frozen_cash = self.state.frozen_cash.max(0.0);` and log warning if pre-clamp value was negative. Consider asserting `previous_frozen >= actual_cost + new_frozen`.
- **Evidence**:
```rust
pub fn settle_buy_fill(&mut self, previous_frozen: f64, actual_cost: f64, new_frozen: f64) {
    let refund = (previous_frozen - actual_cost - new_frozen).max(0.0);
    self.state.frozen_cash -= previous_frozen;
    self.state.frozen_cash += new_frozen;
    self.state.available_cash += refund;
}
```

#### MM-H002 — All REST endpoints lack authentication
- **File**: `moni-rs/crates/moni-api/src/routes.rs` L25-72
- **Kind**: direction-shift
- **Finding**: No auth middleware on any endpoint. Any network-reachable client can place orders, cancel orders, write quotes, advance trade dates, trigger CogAlpha/pipeline compute.
- **Remediation**: Add Bearer token or shared-secret middleware. Gate all mutating routes behind auth. This requires adding middleware layer to Router and propagating auth state.

#### MM-H003 — POST /trade-date/{date} accepts unvalidated date string
- **File**: `moni-rs/crates/moni-api/src/routes.rs` L199-204
- **Kind**: route-fix
- **Finding**: Malformed dates (empty, SQL injection, extremely long strings) flow directly into AccountManager and position rollover.
- **Remediation**: Validate YYYY-MM-DD format before passing to engine.

#### MM-H004 — /agent/action uses **kwargs expansion from untrusted input
- **File**: `local_simulator/api.py` L1371-1397
- **Kind**: route-fix
- **Finding**: `args.pop('rule_id')` mutates input dict; `**args` passes unvalidated keys to engine methods enabling parameter injection.
- **Remediation**: Whitelist expected keys per action instead of `**args`. Validate action against known set.

### MEDIUM

#### MM-M001 — StrategyRegistry Mutex poisoning cascade
- **File**: `moni-rs/crates/moni-strategy/src/strategy.rs` L61
- **Kind**: route-fix
- **Finding**: `.lock().unwrap()` on Mutex; thread panic poisons lock permanently.
- **Remediation**: Use `.lock().unwrap_or_else(|e| e.into_inner())` or parking_lot::Mutex.

#### MM-M002 — StrategyRegistry::get() reconstructs all strategies per call
- **File**: `moni-rs/crates/moni-strategy/src/strategy.rs` L70-87
- **Kind**: structural
- **Finding**: O(n) allocation per lookup; creates default instances ignoring runtime params.
- **Remediation**: Maintain HashMap<String, Arc<dyn BaseStrategy>> populated once in ::new().

#### MM-M003 — SessionStore Mutex poisoning risk
- **File**: `moni-rs/crates/moni-strategy/src/sessions.rs` L48+
- **Kind**: route-fix
- **Finding**: Same Mutex poisoning pattern as MM-M001, affects session lifecycle.
- **Remediation**: Same fix pattern.

#### MM-M004 — MemoryDataProvider RwLock poisoning
- **File**: `moni-rs/crates/moni-data/src/memory.rs` L28,37,44,54,62
- **Kind**: route-fix
- **Finding**: Write-side panic poisons all subsequent reads, crashing engine actor.
- **Remediation**: Use `.unwrap_or_else(|e| e.into_inner())` or parking_lot::RwLock.
