## 项目概述

`api额度监测` 是一个 Windows 桌面 TUI 工具，用 Python（requests + rich + python-dateutil）轮询 `yunyi.cfd`（API 中转/reseller 服务）的 `/user/api/v1/me` 端点，实时展示多个 API Key 的套餐余额/日限额/有效期。入口 `python monitor.py`，依赖 `msvcrt`（Windows 键盘输入）+ `os.system('cls')`。

代码体量小（monitor.py 309 行），但**触及两类敏感路径**（autonomous-constraints §B）：① 凭证/密钥（明文 API key 存 config.json + 作为 Bearer 发出）；② 外部调用（HTTP GET 出站带凭证）。

### 🟠 APIMON-SWALLOW-01 — 裸 except 静默吞所有异常 [MEDIUM]

- **file**: `api额度监测/monitor.py`
- **line**: 48-49, 60-61, 179-180
- **kind**: route-fix（命中 autonomous-constraints「静默吞错」真 bug 例外，非自指 churn）
- **severity**: medium
- **finding**: 三处 `except:` 裸捕获：
  - line 48 `save_config`：写配置失败静默返回 False，调用方（`load_config` line 40 / `process_command` line 268/284）**不检查返回值** → 用户加/删 key 后磁盘满/权限错时无任何反馈，下次启动 key 丢失，用户误以为已保存。
  - line 60 `get_billing_info`：所有网络/解析/连接错一律返回 `"连接错误"`，区分不出 DNS 失败 / TLS 失败 / 401 / 5xx / 超时 / JSON 解析错 —— 排障无抓手。
  - line 179 `parser.parse`：过期时间解析失败静默 `pass`，`expires_display` 留 `"-"`，掩盖 API 返回非 ISO 格式时的格式漂移。
- **evidence**: monitor.py:48 `except:` / :60 `except:` / :179 `except:` 三处裸捕；调用点 line 268/284 未读 `save_config` 返回值。
- **remediation**:
  - line 48 → `except (OSError, json.JSONDecodeError) as e:` + 写 stderr/log
  - line 60 → 拆 `requests.exceptions.Timeout`/`ConnectionError`/`HTTPError`/`json.JSONDecodeError` 分别返回不同 error 串
  - line 179 → `except (ValueError, TypeError) as e:` + 保留 fallback 显示
  - `save_config` 调用点检查返回值，失败时 `state["message"] = "保存失败：磁盘/权限？"`

### 🟡 APIMON-SSRF-01 — base_url 无校验，Bearer 凭证可发往任意主机 [LOW]

- **file**: `api额度监测/monitor.py`
- **line**: 51-56（`get_billing_info`）
- **kind**: route-fix
- **severity**: low
- **finding**: `base_url` 直接来自 config.json（用户可编辑），`api_url = f"{base_url}/user/api/v1/me"` 后 `requests.get(..., headers={"Authorization": f"Bearer {api_key}"})`。无 scheme/host 校验：① config 笔误成 `http://` 明文发凭证；② 配置被改/被恶意覆盖为攻击者主机 → **三把 live key 的 Bearer token 直接外泄给攻击者**。`load_config` 默认值 `https://yunyi.cfd` 是安全的，但运行时 `config.get("base_url", ...)` 取的是文件值，无再校验。
- **evidence**: monitor.py:52 `base_url = base_url.rstrip("/")` 仅去尾斜杠；:53 直接拼 URL；:54 Bearer 头。无任何 `urlparse`/scheme/host 检查。
- **remediation**: `get_billing_info` 入口加 `from urllib.parse import urlparse; p=urlparse(base_url); assert p.scheme=="https"; assert p.hostname=="yunyi.cfd"`（或可配置 allowlist）。非 https / 非 allowlist 主机直接拒绝，不发文凭证。

### 🟡 APIMON-RACE-01 — 后台线程写 data_cache 与主线程迭代竞争 [LOW]

- **file**: `api额度监测/monitor.py`
- **line**: 74（写）vs 108（读迭代）
- **kind**: route-fix
- **severity**: low
- **finding**: `update_data_thread`（daemon 线程）每轮循环 `state["data_cache"][key] = {...}`（line 74）；主线程 `make_dashboard` 在 `Live` 10fps 下迭代 `state["data_cache"].get(key, {})`（line 108）。CPython GIL 保证单次 dict get/set 原子，但**遍历期间被另一线程改 dict**在 CPython 仍可能触发 `RuntimeError: dictionary changed size during iteration`（此处用的是 `.get(key)` 单点查询而非遍历，触发概率低，但 del 操作 `del state["data_cache"][removed]`（line 270）发生在主线程，与后台线程的写并发同 dict → 仍有竞态窗口）。更现实的问题是 `state["config"]` 被 `process_command`（主线程）改 keys 列表时，后台线程正在 `for key in keys`（line 69）迭代同一 list → `list changed size during iteration` RuntimeError 实打实可触发。
- **evidence**: monitor.py:67 `keys = config.get("keys", [])` 拿到 list 引用；:69 `for key in keys:` 迭代；主线程 :283 `keys.append(cmd)` / :267 `keys.pop(idx)` 原地改同一 list → 迭代中改大小。
- **remediation**: `update_data_thread` 取 `keys = list(config.get("keys", []))`（快照拷贝）；`make_dashboard` 取 `keys = list(config.get("keys", []))`；`data_cache` 同理 `snapshot = dict(state["data_cache"])` 后迭代。

### 🟡 APIMON-KEYLEN-01 — `len(cmd) > 10` 当作合法 Key 入库 [LOW]

- **file**: `api额度监测/monitor.py`
- **line**: 281
- **kind**: route-fix
- **severity**: low
- **finding**: `process_command` 的兜底分支 `if len(cmd) > 10:` —— 任何 >10 字符的随机串（含误输入的指令、粘贴的 URL、中文句子）都会被当作 key 追加并 `save_config`，污染 keys 列表，每次启动还会用这把「假 key」发一次 Bearer 请求到 yunyi.cfd。无格式校验。看 config.json 真实 key 格式为 `XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX`（5 段 36 字符）。
- **evidence**: monitor.py:281 `if len(cmd) > 10:` 是唯一准入条件；config.json 三把 key 均匹配 `^[A-Z0-9]{8}-[A-Z0-9]{4}-[A-Z0-9]{4}-[A-Z0-9]{4}-[A-Z0-9]{12}$`。
- **remediation**: 改 `KEY_RE = re.compile(r"^[A-Z0-9]{8}-[A-Z0-9]{4}-[A-Z0-9]{4}-[A-Z0-9]{4}-[A-Z0-9]{12}$")`，`if KEY_RE.match(cmd):` 替换 `len(cmd) > 10`。
