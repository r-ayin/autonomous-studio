## Detailed Findings

### ESE-RT-H01: autonomous-loop.sh archive-live divergence (HIGH)

**File**: `自主循环/scripts/autonomous-loop.sh` (archive) vs `autonomous-studio/scripts/autonomous-loop.sh` (live)

Archive lines 14-20 document ESE-M02 fix adding `AUTONOMOUS_MAX_TURNS` and `AUTONOMOUS_ROUND_TIMEOUT` env var knobs. Lines 113-136 implement conditional `--max-turns` and `timeout` wrapping. Live version has neither — comment on line 14 still says old "cloudcli claude 无 --max-iterations flag" (which ESE-M02 explicitly corrected).

Manifest `_manifest.json` entry for `自主循环/autonomous-loop.sh` claims `"live_md5_match": true` with md5 `1d5b7efec4a6e75204b1ef7cc86d4549`. This is factually wrong given the observed diff.

**Impact**: Engines running from live AS lack per-round timeout protection. A hung claude -p invocation blocks the entire loop indefinitely. Archive has the fix but it was never merged back to live.

**Remediation**: Merge archive ESE-M02 changes into live `autonomous-studio/scripts/autonomous-loop.sh`. Regenerate manifest checksums post-merge.

### ESE-RT-H02: studio-precheck.sh shell injection regression (HIGH)

**File**: `心跳预检/scripts/studio-precheck.sh`

Live version (lines 19-25, 43-49):
```bash
CONSECUTIVE=$(python3 -c "
import json, sys
    d = json.load(open('$CAL_FILE'))
```

Archive version:
```bash
CONSECUTIVE=$(CAL_FILE="$CAL_FILE" python3 -c "
import json, os, sys
    d = json.load(open(os.environ['CAL_FILE']))
```

Live interpolates `$CAL_FILE` directly into Python source string. If CAL_FILE path contains single quote or backslash, Python syntax breaks or executes arbitrary code. Archive correctly passes via environment variable.

**Impact**: Path names with special characters (rare but possible in workspace setups) cause silent precheck failure or code execution. Archive is strictly better.

**Remediation**: Replace live studio-precheck.sh with archive version (env-var approach).

### ESE-RT-M01: distill-patterns.py exception handling lag (MEDIUM)

**File**: `模式蒸馏/scripts/distill-patterns.py`

Live version narrowed bare `except Exception` to `(OSError, ValueError, TypeError, UnicodeDecodeError)` in `load_json_robust` (line 42-45) and `(subprocess.SubprocessError, OSError, TimeoutError)` in step2 LLM handler (line 358). These are SCRIPTS-L04 hardening fixes applied to live.

Archive retains bare `except Exception` at both sites, meaning archive silently swallows KeyboardInterrupt/SystemExit/MemoryError.

**Impact**: Archive copy cannot be used as reliable fallback — it has weaker error semantics than live. If live is corrupted and operator restores from archive, they lose SCRIPTS-L04 hardening.

**Remediation**: Backport live SCRIPTS-L04 exception narrowing into archive copy.

### ESE-RT-M02: Manifest checksum verification broken (MEDIUM, structural)

**File**: `_manifest.json`

At least one entry (`自主循环/autonomous-loop.sh`) claims `live_md5_match: true` while files demonstrably differ. The manifest was generated at `2026-07-03T12:21:52Z` per `extraction_timestamp`, but ESE-M02 fix comments reference `audit-2026-07-03-006` which completed at `2026-07-03T12:35:00Z` — 13 minutes AFTER extraction. The fix was applied to archive post-manifest-generation without updating checksums.

**Impact**: Integrity verification tooling relying on manifest gives false confidence. Downstream consumers (engine-skills-extracted installers, CI checks) trust stale data.

**Remediation**: Add post-fix manifest regeneration step to ESE packaging workflow. Structural debt: register as SD-xxx in structural-debt.md.

### ESE-RT-L01: auto-commit.py push race (LOW)

**File**: `自动提交/scripts/auto-commit.py` lines 325-329, 389-411

Autonomous-active check happens at function entry (L325). Push happens ~60 lines later (L394) unconditionally. Window between check and push: if `.autonomous_active` marker is created during commit processing (e.g., concurrent loop restart), push proceeds despite autonomous mode now being active.

**Impact**: Narrow race. Requires precise timing of marker creation during do_commit execution. Low probability but violates documented invariant "自主模式激活时不自动提交".

**Remediation**: Re-check marker before push, or gate push on same condition.

### ESE-RT-L02: stop-completion-gate.py dual output (LOW)

**File**: `完成度门禁/scripts/stop-completion-gate.py` lines 226-228

```python
print(json.dumps({"decision": "block", "reason": reason}, ensure_ascii=False),
      file=sys.stderr)
print(reason)  # ← stdout, leaks into conversation
sys.exit(2)
```

Claude Code Stop hook protocol reads stderr for JSON decision. The stdout `print(reason)` injects plain-text block reason into conversation context as assistant-visible output. This is noise — the JSON on stderr already carries the reason.

**Impact**: Conversation context pollution. Minor token waste per blocked stop.

**Remediation**: Remove stdout print on line 228.
