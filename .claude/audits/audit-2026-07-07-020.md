## High Severity

### H-001 CORS wildcard origin (api.py:106) [route-fix]
`allow_origins=["*"]` permits any origin to make cross-origin API requests. Combined with bearer tokens stored in JS (localStorage), this enables CSRF-like attacks to exfiltrate PII.

```python
app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"],
                   allow_headers=["*"])
```

**Fix**: Restrict to known frontend origins via config. Set `allow_credentials=False`.

### H-002 API key injection via POST /settings/llm (api.py:460) [route-fix]
Accepts arbitrary `api_key` values and stores them in process memory for 4 hours. An attacker with a bearer token can redirect all LLM calls to an attacker-controlled endpoint.

```python
for field in ("model", ... "api_key"):
    val = getattr(body, field)
    if val is not None:
        config.set_override(f"llm.{backend}.{field}", val)
```

**Fix**: Disallow api_key changes via API, or require re-authentication.

### H-003 SSRF via base_url change in POST /settings/llm (api.py:459) [route-fix]
`base_url` can be set to any URL including internal endpoints (cloud metadata, internal services), using the LLM HTTP client as an SSRF proxy.

**Fix**: Validate base_url against allowlist; block RFC1918/loopback/link-local addresses.

### H-004 Silent auth bypass in dev mode (auth.py:29) [direction-shift]
When `PA_API_TOKEN` is not set, all authentication is silently disabled. All PII endpoints become publicly accessible.

```python
if not expected:
    return True  # dev mode: no token = allow all
```

**Fix**: Require explicit `PA_DEV_MODE=1` to enable open access. Refuse to start without token in production. Bind to 127.0.0.1 when no token.

## Medium Severity

### M-001 Confusing /ws/ exemption in auth (auth.py:49) [route-fix]
`/ws/` in `_EXEMPT_PREFIXES` is redundant since WS doesn't go through HTTP middleware, but creates maintenance confusion.

### M-002 Deceptive User-Agent in web scraping (web.py:35) [route-fix]
Impersonates Chrome browser while scraping Bing/Baidu HTML. ToS violation risk; IP blocking risk.

### M-003 SSRF in ApiWebSearcher._generic (web.py:154) [route-fix]
URL built from config `base_url` without validation; can reach internal services if config is attacker-influenced.

### M-004 No rate limiting on LLM-calling endpoints (api.py:324) [route-fix]
POST /chat, /ingest, /distill, /triggers have no rate limits. Enables cost abuse and DoS via rapid LLM calls.

### M-005 Full PII sent to external LLM providers (chat.py:33) [route-fix]
System prompt includes complete persona profile JSON and memory contents, all sent verbatim to third-party LLM APIs.

### M-006 Inconsistent auth in xiaozhi endpoint (xiaozhi_server.py:209) [route-fix]
Bypasses standard `verify_ws_token()` and implements its own auth logic. Duplicated code is harder to maintain.

### M-007 Non-deterministic ID generation via hash() (storage.py:88) [structural]
Uses `abs(hash()) % 10**12` for IDs. `hash()` is non-deterministic across Python runs (PYTHONHASHSEED). Collision causes silent data overwrites via `INSERT OR REPLACE`. Same pattern in add_event:219, add_reminder:254, add_intervention:184.

### M-008 Module-level CONFIG load (config.py:74) [direction-shift]
Config loaded once at import. Env changes after startup are never picked up.

### M-009 Broadcast without lock (ws_manager.py:44) [route-fix]
`broadcast()` iterates `active` connections without holding `_lock`. Race condition possible with concurrent connect/disconnect.

### M-010 API key over HTTP if base_url is non-TLS (llm.py:421) [route-fix]
If base_url is changed to `http://` (via H-003), API keys are transmitted in cleartext.

### M-011 Destructive DELETE /chat-log without confirmation (api.py:349) [route-fix]
Single request permanently deletes all chat history. No soft-delete, no undo.

## Priority Remediation Order

1. **H-004 + L-004**: Auth bypass + public bind = immediate PII exposure risk. Fix together.
2. **H-001**: CORS wildcard. Quick fix, high impact.
3. **H-002 + H-003**: SSRF + key injection via /settings/llm. Restrict allowed fields.
4. **M-004**: Rate limiting on LLM endpoints. Prevents cost abuse.
5. **M-007**: Switch to uuid4() for ID generation. Prevents silent data loss.
6. **L-001**: Fix dead code indentation in asr.py.
