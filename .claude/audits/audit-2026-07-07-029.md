## Detailed Findings

### MONI-PY-001: SSRF via LLM chat proxy (HIGH, route-fix)

**File**: `local_simulator/api.py:1902-1961`

`/llm/chat` accepts `provider_override.base_url` from request body and uses it directly in `urllib.request.Request` (line 1941-1949). No URL allowlist, no scheme/host validation, no SSRF protection.

**Attack vector**: Attacker sets `base_url` to `http://169.254.169.254/latest/meta-data/` or `http://localhost:PORT/` to probe internal services. The `api_key` is sent in the `Authorization` header (line 1947), so hitting an attacker-controlled URL also exfiltrates credentials.

**Evidence**: 
```python
# api.py:1914
base_url = override.get("base_url") or cfg.get("base_url") or "https://api.deepseek.com/v1"
# api.py:1941-1949
url = base_url.rstrip("/") + "/chat/completions"
req = urllib.request.Request(url, ..., headers={"Authorization": f"Bearer {api_key}"})
with urllib.request.urlopen(req, timeout=120) as resp: ...
```

**Remediation**: Add URL allowlist for base_url (only known LLM provider URLs). Validate scheme is https. Resolve DNS and reject private/reserved IPs (169.254.x.x, 10.x, 172.16-31.x, 127.x, ::1).

### MONI-PY-002: Unbounded backtest frames (MEDIUM, route-fix)

**File**: `local_simulator/api.py:114-127, 1201-1222`

`BacktestPayload.frames` is `List[BacktestFramePayload]` with no max length constraint. Each frame contains a quotes list. `BacktestRunner.run()` creates a full TradingEngine + SQLite DB per run — no internal cap.

**Impact**: Memory exhaustion DoS. Mirrors MONI-API-002 from audit-028 (Rust side) but unfixed in Python.

**Remediation**: Add `Field(max_length=500)` to `BacktestPayload.frames`. Server-side guard: `if len(payload.frames) > MAX_BACKTEST_FRAMES: raise HTTPException(400)`.

### MONI-PY-003: Synchronous blocking WFO on async event loop (MEDIUM, route-fix)

**File**: `local_simulator/api.py:1737-1764`

`/validation/grid-search` calls `wfo_engine.run_wfo()` synchronously inside a FastAPI sync endpoint. WFO involves Monte Carlo simulation and multi-window optimization — CPU-heavy work that blocks the entire uvicorn event loop for potentially minutes. All other requests (WebSocket pushes, health checks, order submissions) are stalled.

Contrast with `/wfo/run` (line 1658-1689) which correctly offloads to `AsyncRunStore`.

**Remediation**: Refactor to use AsyncRunStore pattern (same as `/wfo/run`): wrap in `_target`, call `app.state.run_store.start('grid_search', _target)`, return `run_id`.

### MONI-PY-004: Agent action whitelist bypass (MEDIUM, route-fix)

**File**: `local_simulator/api.py:1355-1386, 1396-1437`

`_filter_action_args` returns args UNFILTERED when action is not in `_ACTION_KWARGS_WHITELIST` (`if allowed is None: return args` at line 1378-1379).

**Unwhitelisted actions**: `place_order`, `cancel_order`, `set_data_source`, `update_settings`, `delete_rule`, `delete_strategy_session`, `evaluate_rules`, `advance_trade_date`.

**Critical path**: `update_settings` (line 1403) passes `args.get('updates', {})` to `engine.update_runtime_settings()` which calls `_deep_update` — a recursive dict merge with NO key validation. Attacker can inject arbitrary config keys (e.g., `{'storage': {'sqlite_path': '/tmp/evil.db'}}`) to corrupt engine state or redirect storage paths.

**Remediation**: (1) Add ALL dispatched actions to `_ACTION_KWARGS_WHITELIST`. (2) In `update_runtime_settings`, validate that updates only contain known config subtrees and reject `__*` internal keys.

### MONI-PY-005: LLM API key leaked via /agent/state/full (HIGH, route-fix)

**File**: `local_simulator/api.py:1328-1341, 1343-1347`

`/llm/config` GET correctly redacts api_key (returns `has_api_key: bool`). But:
- `/agent/state/full` (line 1340) returns `settings: app.state.engine.config` — the FULL config dict including `config['llm']['api_key']` in plaintext.
- `/agent/settings` (line 1343-1347) returns `engine.get_runtime_settings()` which returns `self.config` verbatim (engine.py:135-138).

**Remediation**: Create `sanitize_config(config)` helper that deep-copies and replaces `api_key` values with `'***'`. Apply in both endpoints.

### MONI-PY-006: No authentication on any endpoint (MEDIUM, structural)

**File**: `local_simulator/api.py:920-2003`

All 50+ endpoints including state-mutating ones have zero auth. Default config binds to `127.0.0.1:8000` which limits network exposure, but any local process can fully control the trading engine. Equivalent to MONI-API-003 from audit-028.

**Queued as structural debt SD-014.**

Accepts any WebSocket connection without auth. Pushes real-time trading events. Covered under MONI-API-003 umbrella; noting separately.

**Queued as structural debt SD-015.**
