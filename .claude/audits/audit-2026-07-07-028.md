## High Severity

### H-001 (COGALPHA-01): DeepSeek HTTP calls have no timeout → indefinite hang

- **File:** `moni-rs/crates/moni-cogalpha/src/llm.rs:108`
- **Kind:** route-fix
- **Evidence:** `client: reqwest::Client::new()` — bare client with no connect/request timeout. `complete()` at line 137 `.send().await` blocks indefinitely if remote accepts TCP but never responds. In `run_all_agents`, 21 concurrent tasks each call this; one hung connection stalls the entire round forever.
- **Remediation:** Build client with `reqwest::Client::builder().connect_timeout(10s).timeout(60s).build()`. Accept timeout via config/env.

### H-002 (MONI-API-001): Unbounded Monte Carlo simulations — CPU DoS on async runtime

- **File:** `moni-rs/crates/moni-api/src/routes.rs:317-329`
- **Kind:** route-fix
- **Evidence:** `POST /wfo/monte-carlo` accepts attacker-controlled `n_sims` with no upper bound. Runs synchronously inside async handler (no `spawn_blocking`). Also no `DefaultBodyLimit` configured — `daily_returns` array can be arbitrarily large.
- **Remediation:** Clamp `n_sims.min(10_000)`, clamp `daily_returns.len()`, wrap in `spawn_blocking`, add `DefaultBodyLimit::max(2MB)`.

## Medium Severity

### M-001: avg_cost includes buy-side fees → misleading unrealized PnL

- **File:** `moni-rs/crates/moni-engine/src/position.rs:93-101`
- **Kind:** direction-shift
- **Evidence:** `apply_buy_trade` receives `total_cost = price * qty + fee.total`, inflating `avg_cost`. `mark()` uses this for `unrealized_pnl = (last_price - avg_cost) * qty`, which is understated immediately after purchase. Cash accounting IS consistent (cash-consistent), but display/threshold triggers are misleading. Deviates from standard brokerage practice.
- **Remediation:** Pass `price * qty` (ex-fees) to `apply_buy_trade`; track fees separately. Or document explicitly and adjust `unrealized_pnl`.

### M-002 (M-003): update_quotes matching loop uses stale cloned orders

- **File:** `moni-rs/crates/moni-engine/src/engine.rs:463-490`
- **Kind:** route-fix
- **Evidence:** Clones all open orders into local Vec, iterates calling `attempt_match` on each clone. Two simultaneous resting orders on same symbol could both fill against same `available_volume` because the cloned quote snapshot doesn't reflect volume consumed by earlier fills in the same loop iteration.
- **Remediation:** Re-fetch quote from `data_provider` inside loop, or pass `&Quote` reference to live data.

### M-003 (M-004): settle_buy_fill silently swallows refund when drift-clamp fires

- **File:** `moni-rs/crates/moni-engine/src/account.rs:87-111`
- **Kind:** structural
- **Evidence:** When `frozen_cash` drifts negative beyond tolerance, clamps to 0.0. But `refund` was computed BEFORE clamp as `(prev - cost - new).max(0.0)` = 0.0 in drift scenario. Available cash ends up higher than it should by drift amount. Invariant `available + frozen + market_value = total_assets` violated. Sub-penny impact but breaks accounting identity.
- **Remediation:** After clamp, compute shortfall and deduct from `available_cash`. Or use integer cents arithmetic.

### M-004 (COGALPHA-02): No rate limiting or concurrency cap on LLM calls

- **File:** `moni-rs/crates/moni-cogalpha/src/agents.rs:114-138`
- **Kind:** structural
- **Evidence:** `run_all_agents` fires 21 `tokio::spawn` tasks concurrently with zero semaphore/rate-limiter/token-budget. Each requests `max_tokens=1500`. Full run can issue 63+ concurrent requests. Nothing prevents runaway costs from misconfigured loops.
- **Remediation:** Add `tokio::sync::Semaphore` (permit=5). Add token-counter checked before each call. Integrate leaky-bucket rate limiter.

### M-005 (COGALPHA-03): Prompt injection via user-controlled context fields

- **File:** `moni-rs/crates/moni-cogalpha/src/agents.rs:41-67`
- **Kind:** direction-shift
- **Evidence:** `Agent::build_prompt` interpolates `context["universe"]`, `context["factor_stats"]`, `context["seed_expressions"]` directly into prompt with no sanitization. Attacker controlling context JSON can inject system-level instructions to manipulate factor generation.
- **Remediation:** Validate/sanitize context fields (cap length, strip control chars). Use structured message roles (system vs user) so model distinguishes trusted instructions from untrusted data.

### M-006 (COGALPHA-04): Silent fallback to Mock LLM masks production misconfiguration

- **File:** `moni-rs/crates/moni-cogalpha/src/llm.rs:163-179`
- **Kind:** direction-shift
- **Evidence:** When `MONI_LLM_PROVIDER=deepseek` but `DEEPSEEK_API_KEY` missing, logs `tracing::warn!` and silently returns `MockLlmClient`. System appears operational but generates deterministic fake factors that pass quality checks and enter factor registry. Silent data integrity failure.
- **Remediation:** Make fallback configurable: production mode returns `Err`. Add `MONI_LLM_ALLOW_MOCK_FALLBACK` env var. Emit metric/alert, not just log.

### M-007 (MONI-API-002): Unbounded replay/backtest frame ingestion — memory exhaustion

- **File:** `moni-rs/crates/moni-api/src/routes.rs:239-263, 297-313`
- **Kind:** route-fix
- **Evidence:** `POST /replay/sessions` and `POST /backtest/run` accept unbounded `frames: Vec<ReplayFrameInput>`. Single request with millions of frames consumes unbounded heap during deserialization. Sessions stored in-memory HashMap with no eviction policy.
- **Remediation:** Reject if `frames.len() > MAX_FRAMES` (10k). Add session count cap with LRU eviction. Move heavy computation to `spawn_blocking`. Set global `DefaultBodyLimit`.

### M-008 (MONI-API-003): No authentication/authorization on any endpoint

- **File:** `moni-rs/crates/moni-api/src/routes.rs:25-76, main.rs:80-99`
- **Kind:** structural
- **Evidence:** Every endpoint including state-mutating ones (`POST /order`, `DELETE /order/:id`, `POST /quotes`, `POST /cogalpha/run`) exposed without auth middleware. Server binds via `axum::serve` with no TLS, no bearer token, no API key. Zero `.layer(...)` calls for auth. If reachable beyond localhost, any peer can place orders, overwrite market data, trigger expensive computations.
- **Remediation:** If localhost-only: document + bind `127.0.0.1` default. If exposed: add auth middleware layer. Separate read-only vs mutating routes.

### M-009 (MC-001): Symlink TOCTOU on new-file leaf in resolve_storage_path

- **File:** `moni-rs/crates/moni-core/src/paths.rs:65-81`
- **Kind:** route-fix
- **Evidence:** When leaf file doesn't exist (first-run DB creation), falls back to canonicalizing only parent directory + re-appending filename. Correctly prevents `../` traversal but NOT symlink-based escape at leaf level. Attacker creating symlink at `base_dir/data/sim.db → /etc/shadow` between validation and file-open will write to arbitrary location. Residual gap in 478e4fc fix.
- **Remediation:** After creating/opening file, re-canonicalize full path and re-check `starts_with(base_canon)`. Or use `O_NOFOLLOW` flag. Or create with temp name + atomic rename.

### M-010 (MC-002): Six env-var path functions lack any validation

- **File:** `moni-rs/crates/moni-core/src/paths.rs:8-43`
- **Kind:** structural
- **Evidence:** `resolve_db_path`, `resolve_qlib_provider_uri`, `resolve_baostock_source_dir`, `resolve_output_dir`, `resolve_factor_library`, `resolve_intraday_features_pt` return raw `PathBuf` from env vars with no canonicalization, no base-dir check, no traversal guard. Re-exported as public API. Setting `MONI_STOCK_DB=/etc/cron.d/pwned` silently writes to arbitrary path. MR-M13 fix only covers `resolve_storage_path`.
- **Remediation:** Deprecate/remove if vestigial (only used in tests). Or add `base_dir` parameter + same canonicalize+starts_with check. Document that callers MUST validate.

## Low Severity

### L-001 (M-005): Daily loss limit uses `<=` — blocks at exact threshold

- **File:** `moni-rs/crates/moni-engine/src/risk.rs:62`
- **Kind:** route-fix
- **Evidence:** `account.daily_pnl <= -daily_loss_limit` blocks buys when exactly at limit. Differs from typical `<` convention. Consistent across `pre_trade_check` and `revalidate_at_fill`.
- **Remediation:** Confirm intended semantics. If "at limit" should allow trading, change to `<`.

### L-002 (M-006): release_cash silently caps release to frozen_cash

- **File:** `moni-rs/crates/moni-engine/src/account.rs:70-78`
- **Kind:** structural
- **Evidence:** `amount.min(self.state.frozen_cash)` prevents negative frozen_cash but hides double-release bugs. Second call succeeds with amount=0, returning `Ok(())`.
- **Remediation:** Log warning when `amount > frozen_cash`, or return error/bool indicating partial release.

### L-003 (COGALPHA-05): Unbounded LLM output parsing

- **File:** `moni-rs/crates/moni-cogalpha/src/llm.rs:183-204`
- **Kind:** route-fix
- **Evidence:** `parse_factor_expressions` extracts every matching line with no cap on returned vector size. Hallucinating LLM could return thousands of lines, flooding quality evaluation.
- **Remediation:** Cap `out.len()` at 20 per call. Truncate with debug log. Cap input string length.

### L-004 (MC-003): Config read TOCTOU between canonicalize and read

- **File:** `moni-rs/crates/moni-core/src/config.rs:506-513`
- **Kind:** direction-shift
- **Evidence:** `canonicalize()` derives `base_dir`, then `read_to_string(path)` reads from original path. Narrow window for config replacement between calls. Low risk since config.yaml is admin-controlled.
- **Remediation:** Read from `abs` (canonicalized path) instead of `path` when canonicalize succeeds.

### L-005 (MC-004): No upper-bound sanity checks on fees/ratios in config validation

- **File:** `moni-rs/crates/moni-core/src/config.rs:373-495`
- **Kind:** direction-shift
- **Evidence:** Validation rejects negatives and ratios > 1.0 but not absurd magnitudes. `commission_rate=0.99` or `initial_cash=1e18` pass validation. Defense-in-depth gap.
- **Remediation:** Add soft warnings for out-of-typical-range values via `tracing::warn!`.
