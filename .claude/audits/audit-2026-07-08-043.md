## Findings

### AE-H-001: save-checkpoint.py `run()` shell=True 命令注入 (HIGH)

- **File**: `.claude/hooks/save-checkpoint.py`
- **Line**: 33-34
- **Severity**: HIGH
- **Kind**: route-fix

```python
return subprocess.check_output(
    cmd, shell=True, cwd=PROJECT_DIR, text=True, encoding="utf-8", timeout=timeout
).strip()
```

`run()` 函数接受字符串 `cmd` 并通过 `shell=True` 执行。虽然当前调用点（L55-57）只传硬编码字符串 `"git branch --show-current"` / `"git status --short"` / `"git log -1 --format=\"%h %s\""`，但该函数是模块级公开工具函数，任何未来修改若插入用户/环境变量到 cmd 字符串即触发注入。

**实际风险**：中等——当前无外部输入流入 cmd，但设计不安全。同项目 watchdog.sh L39 也有类似模式（f-string 嵌入 `$LATEST` 路径到 python3 -c），但路径来自硬编码变量。

**Remediation**: 将 `run()` 改为 list-form `subprocess.check_output(["git", "branch", "--show-current"], ...)` 去掉 `shell=True`。Windows `%` 转义逻辑可移除（list form 不需要）。

### AE-H-002: watchdog.sh f-string 嵌入文件路径到 python3 -c (HIGH)

- **File**: `.claude/watchdog.sh`
- **Line**: 36-44
- **Severity**: HIGH
- **Kind**: route-fix

```bash
CHECKPOINT_TS=$(python3 -c "
import json
try:
    with open('$LATEST') as f: d = json.load(f)
    ts = d.get('timestamp', '')
    print(ts if ts and ts != 'null' and ts != 'None' else '')
except Exception:
    print('')
" 2>/dev/null)
```

`$LATEST` 通过 bash 变量展开直接嵌入 Python 源码字符串。虽然 `$LATEST` 当前来自硬编码 `PROJECT_DIR` + 固定后缀，但若 PROJECT_DIR 含单引号或特殊字符（如 workspace 路径迁移），会导致 Python 语法错误或非预期代码执行。

**Remediation**: 改用环境变量传递路径：`LATEST_FILE="$LATEST" python3 -c "import os,json; ...open(os.environ['LATEST_FILE'])..."`

### AE-M-001: save-checkpoint.py `write_checkpoint()` 非原子写 (MEDIUM)

- **File**: `.claude/hooks/save-checkpoint.py`
- **Line**: 182-193
- **Severity**: MEDIUM
- **Kind**: route-fix

```python
with open(filepath, "w", encoding="utf-8") as f:
    json.dump(checkpoint, f, ensure_ascii=False, indent=2)
shutil.copy2(filepath, LATEST_LINK)
```

checkpoint 文件和 latest.json 都用直接 `open("w")` 写入，非原子。进程崩溃/断电时可能留下半截 JSON → 下次 `resume-checkpoint.py` 读到损坏文件。对比 `incremental-save.py` L42-57 已正确使用 tempfile+fsync+os.replace 原子写。

**Remediation**: 复用 incremental-save.py 的 atomic write pattern（tempfile → fsync → os.replace）。

### AE-M-002: decision-observer.py `write_session_context()` TOCTOU race (MEDIUM)

- **File**: `.claude/hooks/decision-observer.py`
- **Line**: 172-185
- **Severity**: MEDIUM
- **Kind**: route-fix

```python
if os.path.exists(filepath):
    with open(filepath, "r", ...) as f:
        existing = json.load(f)
existing.update(data)
with open(filepath, "w", ...) as f:
    json.dump(existing, ...)
```

Read-modify-write 无锁。两个并发 Stop hook 可能丢失一个的更新。session JSON 是非关键诊断数据，丢失影响低，但模式不正确。

**Remediation**: 用 `fcntl.flock` 或 tempfile+rename 原子替换。

### AE-M-003: resume-checkpoint.py 固件硬编码 Windows 路径 E:/x-tool (MEDIUM)

- **File**: `.claude/hooks/resume-checkpoint.py`
- **Line**: 122, 124
- **Severity**: MEDIUM
- **Kind**: route-fix

```python
- 读 `E:/x-tool/.claude/decision-log.jsonl` 最后 5 行
- 首条回复 → 读 `E:/x-tool/.claude/memory/autonomous-state.md`
```

`build_engine_firmware()` 输出的指令中硬编码了 Windows 路径 `E:/x-tool/`。该 repo 已在 Linux workspace 运行（git history f4ea72d 曾修 memory 路径但未修 firmware 模板）。每次 SessionStart 注入的固件指令指向不存在的路径 → 引擎恢复流程失效。

**Remediation**: 用 `{PROJECT_DIR}` 替换硬编码路径（L112 的 f-string 已对部分路径做了此处理，但 L122/124 遗漏）。

### AE-L-001: protocol-check.py subprocess env 继承泄露 (LOW)

- **File**: `.claude/hooks/protocol-check.py`
- **Line**: 72
- **Severity**: LOW
- **Kind**: route-fix

```python
env={**os.environ, "CLAUDE_PROJECT_DIR": WORKSPACE_ROOT}
```

bootstrap subprocess 继承全部父进程环境变量（含可能的 API keys、tokens）。bootstrap.py 是受信任本地脚本，风险极低，但最佳实践应白名单传递。

**Remediation**: 仅传递必要 env vars：`{"PATH": ..., "PYTHONPATH": ..., "CLAUDE_PROJECT_DIR": ...}`

### AE-L-002: notify-phone.py ntfy message 未限制长度 (LOW)

- **File**: `.claude/hooks/notify-phone.py`
- **Line**: 96-103
- **Severity**: LOW
- **Kind**: route-fix

curl `-d message` 发送通知内容无长度上限。ntfy.sh 免费层限制 4KB body。超长消息被服务端截断但不报错给调用方。

**Remediation**: `message = message[:2000]` 截断。
