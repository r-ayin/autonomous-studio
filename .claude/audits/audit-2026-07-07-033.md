## Findings

### H-META-001 — derived_fixes non-terminal status blocks cycle semantics

- **file**: `.claude/audit-cycle-state.json`
- **line**: 27 (M-POST-001 entry `"status": "deferred-to-debt-queue"`)
- **severity**: high
- **kind**: route-fix
- **finding**: The `derived_fixes[].status` field uses value `deferred-to-debt-queue`, which is NOT in the documented terminal-state enum `{pending, merged, rejected}` (constraints §A line 54). The cycle-complete transition predicate ("全部 merged|rejected 后 status=cycle-complete") cannot be mechanically verified when non-enum values are present. Current `pending_count: 0` was achieved by manual override rather than state-machine convergence. Any future automation that checks `derived_fixes.every(f => ['merged','rejected'].includes(f.status))` will return false despite cycle being declared complete.
- **remediation**: Replace `deferred-to-debt-queue` with `rejected` (the closest semantic match — the fix was not merged, it was deferred to a different tracking system). Add `_deferred_to_debt_id` metadata field to preserve traceability. This restores enum conformance without losing information.
- **evidence**: Grep of audit-cycle-state.json shows exactly one occurrence of `deferred-to-debt-queue`. Constraints §A line 54 explicitly enumerates `{pending, merged, rejected}`. No other derived_fixes entry uses a non-enum status.
