## Summary

personal-assistant 是一个结构良好的 MVP 阶段个人助理项目，具备反幻觉验证管线、可插拔 LLM 后端和确定性时间解析。但存在多个严重问题：**.env 明文 API key 提交到仓库**、**通配符 CORS + 零认证暴露所有破坏性端点**、**路径遍历漏洞允许任意文件写入**、**全局可变计数器导致线程安全问题**、**ASR 模块死代码**，以及**全项目零 audit-log 埋点**（违反 autonomous-constraints.md B 节所有敏感路径）。核心管线逻辑健全且 stub 后端测试良好，但安全与生产就绪有显著缺口。

## Findings

### H-001 severity: high kind: route-fix
- **file**: .env
- **line**: 1-7
- **finding**: API keys stored in plaintext in `.env` which is not gitignored. Contains `ANTHROPIC_AUTH_TOKEN=TOKEN-REDACTED`, `DEEPSEEK_API_KEY=sk-REDACTED`. Real credentials visible to anyone with repo access.
- **remediation**: Add `.env` to `.gitignore`. Rotate exposed keys immediately. Use secrets manager or ensure `.env` is only created locally.
- **data_flow_context**: `.env` -> `config.load_env()` -> `config._substitute()` -> injected into `llm.py` HTTP calls as Bearer/x-api-key headers.

### H-002 severity: high kind: route-fix
- **file**: personal_assistant/api.py
- **line**: 15-16
- **finding**: CORS set to `allow_origins=["*"], allow_methods=["*"], allow_headers=["*"]`. Combined with zero authentication on all endpoints, any webpage can call destructive endpoints (DELETE /chat-log, POST /distill, POST /settings/llm, POST /inbox/upload).
- **remediation**: Restrict `allow_origins` to actual frontend origins. Add authentication middleware for sensitive endpoints.
- **data_flow_context**: Any cross-origin request -> POST /chat (stores user data), DELETE /chat-log (deletes data), POST /settings/llm (inject attacker-controlled api_key/base_url).

### H-003 severity: high kind: direction-shift
- **file**: personal_assistant/api.py
- **line**: 248-268 (POST /settings/llm)
- **finding**: `/settings/llm` accepts `api_key` and `base_url` in POST body with no authentication. Attacker can redirect all LLM calls to malicious server by setting `base_url`; `api_key` will be sent to it. Combined with H-002 (wildcard CORS), exploitable from any webpage.
- **remediation**: Add authentication. At minimum restrict to localhost-only or add bearer token. Consider not accepting `base_url` changes via HTTP API.
- **data_flow_context**: Attacker-controlled base_url -> `config.set_override()` -> `get_llm()` reads override -> all subsequent LLM calls send user data + api_key to attacker's server.

### H-004 severity: high kind: route-fix
- **file**: personal_assistant/api.py
- **line**: 271-283 (POST /inbox/upload)
- **finding**: Path traversal vulnerability. `filename` query parameter used directly in `dest = inbox / filename`. Filename like `../../config/default.json` writes arbitrary files outside inbox directory. Only check is `.endswith((".txt", ".srt"))`, satisfied by `../../evil.txt`.
- **remediation**: Sanitize filename: `filename = Path(filename).name`. Verify `dest.resolve().is_relative_to(inbox.resolve())`.
- **data_flow_context**: User HTTP request -> `filename` query param -> `inbox / filename` -> `dest.write_bytes(content)` -> arbitrary file write.

### M-001 severity: medium kind: route-fix
- **file**: personal_assistant/verify.py
- **line**: 17-18
- **finding**: `_item_counter` is module-level global integer for verification item IDs. Reset to 0 in `run_all()` (line 218). In concurrent environment (multiple POST /verify), race condition produces duplicate IDs and corrupted reports.
- **remediation**: Use thread-local counter, pass counter state through parameters, or use `uuid.uuid4()`.
- **data_flow_context**: Multiple concurrent POST /verify -> `run_all()` -> reset `_item_counter=0` -> `_next_id()` returns colliding IDs.

### M-002 severity: medium kind: route-fix
- **file**: personal_assistant/asr.py
- **line**: 208-224
- **finding**: Dead code: `scan_once()` and `run_loop()` defined inside `query_habits()` after `return` statement on line 206, making them unreachable. Likely intended as `IngestionPipeline` class methods but indented under wrong scope.
- **remediation**: Move back into `IngestionPipeline` class (after `_analytics` ~line 164), or delete if `ingest.scan_inbox()` replaced functionality.

### M-003 severity: medium kind: route-fix
- **file**: personal_assistant/api.py
- **line**: 161-163 (DELETE /chat-log)
- **finding**: No authentication or confirmation on DELETE /chat-log. Combined with wildcard CORS (H-002), any webpage can silently delete all chat history containing user conversations (PII).
- **remediation**: Add authentication or require confirmation token.

### M-004 severity: medium kind: route-fix
- **file**: personal_assistant/storage.py
- **line**: 52-56
- **finding**: `connect()` runs `executescript(SCHEMA)` (full DDL with CREATE TABLE IF NOT EXISTS) on every database connection. Redundant overhead for every API call. While `IF NOT EXISTS` prevents errors, executing 15+ CREATE statements per connection is wasteful.
- **remediation**: Add module-level `_schema_initialized = False` flag; run SCHEMA once per process lifetime.

### M-005 severity: medium kind: structural
- **file**: personal_assistant/storage.py
- **line**: 52-56, 88, 184, 219, 253, etc.
- **finding**: Every storage function opens/closes own DB connection via `with connect() as c:`. No connection pooling or reuse. Excessive overhead for concurrent web requests; prevents proper transaction management.
- **remediation**: Implement connection pool or per-request connection pattern (FastAPI dependency injection).

### M-006 severity: medium kind: route-fix
- **file**: personal_assistant/llm.py
- **line**: 301-305
- **finding**: `_post_json()` does not handle network errors (timeout, connection refused, DNS failure) with retry or meaningful error message. `urllib.request.urlopen` raises opaque exception propagating unhandled through entire pipeline.
- **remediation**: Wrap in try/except with specific error types (URLError, HTTPError, timeout), log error, return structured error or raise custom exception.

### M-007 severity: medium kind: route-fix
- **file**: personal_assistant/storage.py
- **line**: 88
- **finding**: Memory IDs generated using `abs(hash(...)) % 10**12`. Python `hash()` is non-deterministic across processes (PYTHONHASHSEED since 3.3). Same content ingested in different runs produces different IDs, breaking deduplication. Same issue in `add_intervention` (L184), `add_event` (L219), `add_reminder` (L254), `add_wiki_page` (L379).
- **remediation**: Use deterministic hash: `hashlib.sha256(content.encode()).hexdigest()[:12]` or UUID.

### M-008 severity: medium kind: route-fix
- **file**: personal_assistant/api.py
- **line**: 246-268 (POST /settings/llm)
- **finding**: Runtime override stores api_key in `config._RUNTIME` dict in plaintext in process memory with no expiration. Response confirms `api_key` was accepted via `applied` list.
- **remediation**: Consider not accepting api_key via HTTP. If needed, store encrypted or don't confirm in response.

### L-001 severity: low kind: route-fix
- **file**: personal_assistant/web.py
- **line**: 23-28
- **finding**: `_fetch()` spoofs Chrome User-Agent to scrape Bing/Baidu search results. Violates ToS; may result in IP blocking. No rate limiting.
- **remediation**: Document limitation. Consider official search APIs or make User-Agent configurable.

### L-002 severity: low kind: route-fix
- **file**: personal_assistant/config.py
- **line**: 12-22
- **finding**: `.env` parser doesn't handle multi-line values, values with `#` (comment char), or escaped quotes. API key containing `#` would be truncated.
- **remediation**: Use robust parser or document limitations. Handle `#` within quoted values.

### L-003 severity: low kind: route-fix
- **file**: personal_assistant/ingest.py
- **line**: 78-99 (scan_inbox)
- **finding**: `scan_inbox()` calls `inbox.iterdir()` which raises `FileNotFoundError` if inbox doesn't exist. API endpoint POST /ingest (api.py:84) calls directly without ensuring dirs exist.
- **remediation**: Add `inbox.mkdir(parents=True, exist_ok=True)` at start of `scan_inbox()`.

### L-004 severity: low kind: route-fix
- **file**: personal_assistant/reminders.py
- **line**: 100-101
- **finding**: Recurring reminder reschedule appends `-next` to ID. Over multiple firings: `rm-123-next-next-next...`. Unbounded growth causes performance issues.
- **remediation**: Generate fresh ID or keep same ID and update `when_dt` + reset `fired=0`.

### L-005 severity: low kind: route-fix
- **file**: personal_assistant/storage.py
- **line**: 341-352 (add_chat_log)
- **finding**: Schema migration catches bare `Exception` to detect missing `evidence` column. If INSERT fails for other reasons (disk full, locked DB), real error swallowed and ALTER TABLE also fails silently.
- **remediation**: Catch `sqlite3.OperationalError` specifically; check error message for "no column named evidence".

### I-001 severity: info kind: route-fix
- **file**: personal_assistant/temporal.py
- **line**: 133
- **finding**: `resolve_range("本月")` computes next month as `start.month + 1` = 13 for December. Passed to `_safe_date()` which returns None. Fallback `_safe_date(start.year + 1, 1, 1)` handles correctly but logic unnecessarily convoluted.
- **remediation**: Directly compute January 1st of next year when month is 12.

## Structural Debt Candidates

- **SD-001**: No authentication layer — entire API has zero auth. Adding requires middleware design + token management + all endpoint updates. Direction-shift affecting entire api.py + frontend.
- **SD-002**: Connection-per-call storage pattern — every function opens own connection + re-runs DDL. Refactoring to pooling touches 30+ functions.
- **SD-003**: No audit-log infrastructure — zero logging exists. Requires new module + instrumenting 14+ paths across 6+ files.
- **SD-004**: Global mutable state in verify.py (`_item_counter`) and config.py (`_RUNTIME`) — not thread-safe. Fix requires DI or synchronization primitives.

## Verified Safe Patterns

1. SQL injection protection — all SQLite queries use parameterized `?` placeholders
2. LLM output parsing — `extract_json()` safely handles malformed output
3. Anti-hallucination pipeline — verify.py -> temporal.py enforces deterministic time + source grounding
4. Config env substitution — `_substitute()` has loop limit of 10 preventing infinite recursion
5. Embedder normalization — `HashingEmbedder` normalizes to unit L2 norm, handles empty strings
6. Cosine similarity — `search_memories()` adds 1e-9 epsilon preventing division-by-zero
7. File suffix validation — upload checks `.endswith((".txt", ".srt"))` (insufficient alone due to H-004)
8. Persona version history — immutable versions with monotonic numbers
9. Temporal parser — `_safe_date()` correctly handles invalid dates; December overflow has working fallback
