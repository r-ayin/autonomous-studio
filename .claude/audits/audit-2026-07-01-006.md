## Findings

### PA-H-001 [HIGH] api.py:15 — CORS allow_origins=["*"] 暴露所有端点
- **file**: personal_assistant/api.py
- **line**: 15
- **severity**: high
- **kind**: direction-shift (触及公共接口行为；修需改 CORS 配置策略，可能影响前端部署拓扑)
- **finding**: `CORSMiddleware(allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])` 配合无鉴权的 POST /chat /ingest /inbox/upload /settings/llm，使任意恶意网页可跨域调用全部 API。`/inbox/upload?filename=` 还可被 CSRF 注入任意 .txt/.srt 文件到 inbox；`/settings/llm` 可被跨域改 LLM backend/api_key。
- **remediation**: 改为显式 origin 白名单（从 config.default.json 读 `cors.allowed_origins`，默认仅 `http://localhost:*` + 同源）。同时给写端点加最小 CSRF token / Referer 校验。**注：这是公共接口变更，触发 direction-shift worktree**。
- **evidence**: api.py:15-16; 配合 api.py:185-197 (inbox_upload 无 auth)、api.py:162-182 (llm_settings_update 无 auth)。

### PA-H-002 [HIGH] api.py:185-197 — /inbox/upload 路径穿越 + 无大小限制
- **file**: personal_assistant/api.py
- **line**: 185-197
- **severity**: high
- **kind**: route-fix
- **finding**: `filename` 从 query 参数直取，仅校验 `.endswith((".txt",".srt"))`。攻击者可传 `../../etc/cron.d/x.txt` 绕过目录写入任意位置（Path traversal）。且 `await request.body()` 无大小上限，可被 DoS 打爆内存/磁盘。
- **remediation**: ① `Path(filename).name` 剥路径分量；② 校验 dest.resolve().is_relative_to(inbox)；③ body 限长（如 50MB），超限返 413。
- **evidence**: api.py:189-195 `dest = inbox / filename; dest.write_bytes(content)`。

### PA-M-001 [MEDIUM] api.py:75-80,95-97 — /chat + /chat-log 无鉴权暴露 PII
- **file**: personal_assistant/api.py
- **line**: 75-80, 95-97
- **severity**: medium
- **kind**: route-fix
- **finding**: `/chat` POST 接收用户消息、`/chat-log` GET 返回完整对话历史（含 PII），均无鉴权。结合 H-001 CORS *，任意网页可读取用户全部聊天内容。

### PA-M-003 [MEDIUM] storage.py:86-97 — add_memory embedding 未做 shape/dtype 校验
- **file**: personal_assistant/storage.py
- **line**: 86-97, 119-130
- **severity**: medium
- **kind**: route-fix
- **finding**: `embedding.tobytes()` 直接入库，无 dtype/shape 校验；检索时 `np.frombuffer(..., dtype=np.float32)` 假设所有行都是 float32 同维。若 embedder 切换维度或返回 float64，search_memories 会 vstack 失败或算出错误相似度。
- **remediation**: add_memory 前 assert `embedding.dtype==np.float32 and embedding.shape==(dim,)`（dim 从 config 读）；不合规则丢弃或重 embed。
- **evidence**: storage.py:94, 125。

### PA-L-001 [LOW] storage.py:87,158,186,214,301 — ID 生成用 abs(hash(...))%10**12 碰撞风险
- **file**: personal_assistant/storage.py
- **line**: 87, 158, 186, 214, 301
- **severity**: low
- **kind**: route-fix
- **finding**: 多处用 `abs(hash(tuple)) % 10**12` 作主键。Python hash 每进程随机化、12 位十进制空间仅 ~40 bit，高吞吐下碰撞概率非平凡；且 hash(tuple) 对 str 内容敏感，相同输入在不同 PYTHONHASHSEED 下结果不同，导致幂等性破坏。
- **remediation**: 改用 uuid4 或 sha256(content)[:16] hex 作稳定 ID。
- **evidence**: storage.py:87, 158, 186, 214, 301。
