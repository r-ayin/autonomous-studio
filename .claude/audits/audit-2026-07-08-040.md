### H-META-040-001 [HIGH] derived_fixes "merged" 声明虚假 — SD materialization commits 从未 merge 到 main

**位置**: audit-cycle-state.json derived_fixes[0,1] (H-META-039-001, M-META-039-002)
**证据**:
- derived_fixes[0] fix_case_id=case-2026-07-07-107 status="merged"，声称 "materialized 10 phantom SDs via opt-autonomous-1783381217"
- `git branch -a --contains de47ec1` → 仅 `auto/opt-autonomous-1783381217`（worktree branch），不在 main
- `git log main --oneline | grep -c "de47ec1\|eb0a30e\|93872a4\|4948ff1"` → **0**
- `git show main:.claude/structural-debt.md | grep "^## SD-"` → 仅 SD-001..SD-005
- 4 个 SD materialization commits (de47ec1, eb0a30e, 93872a4, 4948ff1) 全部只存在于 worktree branches，从未通过 opt-worktree.sh merge 到 main

**影响**: 
- structural-debt.md 缺失 SD-007, SD-009..SD-017（共 10 条），用户无法看到这些结构性债务
- audit-cycle-state 声称 cycle-complete 基于虚假前提——derived_fixes 标 merged 但实际未 merge
- 引擎状态文件与代码仓库真相脱节，违反 "merged = dispatched+pushed" memory 注释（应改为 "merged = on main"）

**严重度**: HIGH — 状态完整性破坏，跨 context 持久化数据不可信
**Kind**: route-fix
**Remediation**: 
(a) 把 derived_fixes[0,1] status 改回 pending（或 rejected，若 worktree 已 stale）；
(b) 重新 dispatch SD materialization（读 worktree branch 内容 → 新 opt-worktree commit → merge main）；
(c) 修正 memory/audit-cycle-merged-means-dispatched.md："merged" 语义从 "dispatched+pushed" 改为 "verified on main via git log"；
(d) 在 opt-worktree.sh cmd_commit 后加 post-merge verification step（grep main for expected content）。
