## Findings

### PA-POST-M-001 (medium, direction-shift) — android api() fetch 无 AbortController
- **File**: android/index.html:584
- **Evidence**: `const res = await fetch(path, { ...opts, headers });` — 无 signal、无 timeout；后端卡住时 UI 永久 pending
- **Remediation**: android api() 内加 AbortController + 8s timeout（慢端点如 /wiki/build 走白名单），catch AbortError 返 null 或 throw 友好错误
- **Kind rationale**: 跨模块一致性修复（web ↔ android API 客户端契约对齐），触及公共接口行为

### PA-POST-M-002 (medium, structural) — liveBus WebSocket 无 ping/pong 心跳
- **File**: android/index.html:596
- **Evidence**: `ws.onopen = () => { status = 'on'; }; ws.onclose = () => { status = 'off'; schedule(); };` — 无 setInterval ping、无 onmessage 重置 lastSeen
- **Impact**: 移动网络 NAT 静默断开后状态卡在 'on' 假活，用户以为在线实则消息丢失
- **Remediation**: 每 30s send JSON `{type:'ping'}` + server pong；客户端 60s 未见任何帧则主动 close+重连。relay_bridge 已有 PING_INTERVAL=30 但 android 直连 PA 后端 /ws/live 时不经过 relay
- **Kind rationale**: 需 web socket 协议层双向改造（client + server），拆不成单文件 route-fix → structural-debt

### PA-POST-L-001 (low, route-fix) — RELAY_TOKEN 为空时 _check_token 恒返 True
- **File**: scripts/ecs_relay/relay_bridge.py:86
- **Evidence**: `if not RELAY_TOKEN: return True  # dev 放行`
- **Remediation**: prod systemd unit 强制 PA_RELAY_TOKEN 非空；或在 main() 启动时若 host!=127.0.0.1 且 token 空则 refuse to start
- **Kind rationale**: 单文件 1-3 行改动

### PA-POST-L-002 (low, structural) — _origin_ok 在无 Origin header 时放行
- **File**: scripts/ecs_relay/relay_bridge.py:106
- **Evidence**: `if not origin: return True  # 非浏览器客户端不发 Origin，放行（token 是主防线）`
- **Remediation**: 可接受（token 确实为主防线）；若要更严，对 /ws/tunnel 强制要求 Origin 白名单，仅 /ws/live 放宽给 ESP32 固件
- **Kind rationale**: 设计取舍非 bug，structural-debt 队列等待用户授权

### PA-POST-L-003 (low, structural) — /ws/xiaozhi accept 后再校验 token
- **File**: personal_assistant/xiaozhi_server.py:221
- **Evidence**: `if auth.is_auth_enabled() and not auth._check(token): await ws.close(code=1008); return; await ws.accept()`
- **Impact**: 未通过鉴权的连接仍消耗 WS 握手资源（DoS amplification vector）
- **Remediation**: 仿 relay_bridge process_request 模式：在 starlette middleware 或 websocket validator 阶段先校验 token，失败直接发 HTTP 403 不进 WS 协议
- **Kind rationale**: 需重构 WS handler 结构，非局部 fix

### PA-POST-I-001 (info, route-fix deferred) — bootstrap 5s race 超时后 Promise.all 仍在后台跑
- **File**: web/index.html:187
- **Evidence**: `Promise.race([window.PA.bootstrap(), new Promise(r => setTimeout(r, 5000))]);` — 输掉 race 的 bootstrap 继续执行至完成
- **Mitigation already present**: 每个 get 自带 8s AbortController，最多再跑 8s 自动停
- **Strict remediation**: race resolve 后调 ctrl.abort() 取消剩余请求
- **Disposition**: deferred (sub-churn-threshold, 现有 defense-in-depth 已足够)
