## Finding Details

### SH-VIS-M-001: Binary string accumulation O(n²) in image compression paths

**File**: `src/background/vision-adapter.ts` lines 320-322 and 346-348
**Severity**: medium
**Kind**: route-fix

```typescript
let binary = '';
for (let i = 0; i < bytes.length; i++) binary += String.fromCharCode(bytes[i]);
return btoa(binary);
```

This pattern appears in both `preprocessForVL` and `compressFallback`. For a 2500×2500 JPEG at quality 0.9 (~800KB compressed), this loop performs ~800K string concatenations. Each `+=` on a growing string may trigger reallocation; V8 optimizes this to amortized O(n) in many cases but it's not guaranteed, especially in service worker contexts with constrained memory. The same anti-pattern was partially addressed by the >10MB early reject (SH-L-001), but compressed outputs between 1.5MB–10MB still hit this path.

**Remediation**: Replace with chunked approach or `Blob`→`FileReader.readAsDataURL`:
```typescript
const blob = await canvas.convertToBlob({ type: 'image/jpeg', quality: 0.9 });
return new Promise<string>((resolve) => {
  const reader = new FileReader();
  reader.onloadend = () => resolve((reader.result as string).split(',')[1]);
  reader.readAsDataURL(blob);
});
```
Or use chunked `Uint8Array`→base64 conversion (btoa in chunks of 32KB).

### SH-SIDE-L-001: onDeleteText uses array index, inconsistent with id-based deletion

**File**: `src/sidepanel/SidePanelApp.tsx` line 915
**Severity**: low
**Kind**: route-fix

The text mark delete button calls `onDeleteText?.(i)` where `i` is the map index. But `handleRegionsChange` (line 144) correctly uses `VerifiedText.id` for the deleted-stack filter (SH-M-005 fix). If the user deletes a text mark via the overlay button while `result.details` has been reordered or filtered since last render, the wrong item could be deleted. In practice the risk is low because `result.details` is stable between renders, but the inconsistency means the SH-M-005 fix is incomplete.

**Remediation**: Change `onDeleteText` signature to accept `id: string` instead of `idx: number`, or pass `t.id` from the map and look up by id in `handleDeleteText`.
