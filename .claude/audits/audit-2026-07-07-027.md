## New Findings

### M-001: clearTimeout(tid) placed after fetch resolve but before r.json() — body-read phase has no timeout protection
- **File**: web/api.js:55-58
- **Severity**: medium
- **Kind**: structural
- **Evidence**: `const r = await fetch(..., signal: ctrl.signal); clearTimeout(tid); ... return await r.json();` — once headers arrive (fetch resolves), the abort timer is cleared. If the server sends 200 OK then stalls the response body, r.json() hangs indefinitely with no abort signal active. Same pattern at line 71-73 for post().
- **Remediation**: Move clearTimeout after r.json() completes (or use a single AbortController that stays active through body read). Alternatively wrap r.json() in its own timeout race.

### M-002: Bootstrap Promise.race 5s timeout does not cancel in-flight fetches — orphan requests continue consuming resources
- **File**: web/index.html:187-191
- **Severity**: medium
- **Kind**: structural
- **Evidence**: When the 5s timer wins the race, bootstrap()'s internal Promise.all with 7+ concurrent fetches continues running unsupervised. These fetches have their own 8s AbortController timeouts but those only fire at 8s, meaning between 5s–8s there are 7+ in-flight requests whose responses will mutate MockData AFTER root.render has already executed with stale/mock data, triggering an unexpected re-render with partial live data.
- **Remediation**: Pass an AbortSignal into bootstrap() and propagate it to each get() call, so the 5s race loser actually cancels all in-flight fetches.

### M-003: Android api() function has no timeout/AbortController — any fetch can hang forever blocking UI
- **File**: android/index.html:581-588
- **Severity**: medium
- **Kind**: structural
- **Evidence**: `async function api(path, opts = {}) { ... const res = await fetch(path, { ...opts, headers }); ... }` — no AbortController, no timeout. Compare to web/api.js which got AbortController in d6f650b. The android app makes 5+ api() calls on screen load. Any one hanging blocks that screen's rendering indefinitely.
- **Remediation**: Add AbortController with timeout to android api(), matching web/api.js pattern. Use ~10s default.

### M-004: POST /reminders accepts unbounded 'what' string with no length validation
- **File**: personal_assistant/api.py:399-401
- **Severity**: medium
- **Kind**: route-fix
- **Evidence**: `class ReminderIn(BaseModel): what: str; when_raw: str = ""` — Pydantic str with no max_length. A malicious or buggy client can POST a multi-MB 'what' payload stored verbatim in SQLite, enabling resource exhaustion and UI DoS.
- **Remediation**: Add `what: str = Field(min_length=1, max_length=1000)` to ReminderIn. Also truncate in storage.add_reminder as defense-in-depth.

### L-001: PA_TOKEN ASCII regex rejects JWT-format tokens containing dots
- **File**: web/api.js:23, android/index.html:574
- **Severity**: low
- **Kind**: direction-shift
- **Evidence**: `/^[A-Za-z0-9_-]+$/` accepts hex tokens but rejects JWT format (header.payload.signature with dots). If backend migrates to JWT/OIDC, frontend silently strips tokens and operates unauthenticated.
- **Remediation**: Add dot to allowed charset: `/^[A-Za-z0-9._-]+$/`. Or document constraint explicitly.

### L-002: postMessage with wildcard '*' origin leaks tweak state to any parent frame
- **File**: android/index.html:347, 399
- **Severity**: low
- **Kind**: structural
- **Evidence**: `window.parent.postMessage({ __edit_mode_set_keys: state }, '*');` — if embedded in malicious iframe, parent receives full tweaks state. Incoming handler doesn't validate event.origin.
- **Remediation**: Replace '*' with specific expected parent origin; validate event.origin in listener.

### L-003: WS reconnect backoff caps at 30s but retry counter never resets on transient recovery
- **File**: android/index.html:602
- **Severity**: low
- **Kind**: structural
- **Evidence**: After 9 consecutive failures, reconnect stops permanently even if network recovers. No UI indication that reconnection has been abandoned.
- **Remediation**: Reset retry on navigator.onchange or add periodic probe. Emit status event when retry > 8.

### L-004: Playwright-style tests are actually TestClient API-level tests — naming mismatch
- **File**: tests/test_ui_click_flow.py:1-5
- **Severity**: low
- **Kind**: direction-shift
- **Evidence**: Docstring says "dev 盒无头less chrome，无法真浏览器点击；改为 TestClient". All 9 tests use FastAPI TestClient, not playwright. Commit af5c873 claims "playwright chromium headless" but committed tests don't do that.
- **Remediation**: Rename to test_api_click_flow.py or clarify in comments. Ensure real playwright tests are in CI if they exist.

### I-001: WS token passed as URL query parameter — visible in access logs
- **File**: android/index.html:595
- **Severity**: info
- **Kind**: structural
- **Evidence**: Unavoidable for browser WebSocket API. Token is properly encodeURIComponent'd. Risk mitigated by hex-only format and HTTPS transport.
- **Remediation**: Document in security notes. Consider server-side log scrubbing.
