## Executive Summary

personal-assistant 是"数据完全本地"的全自动个人助手：24h 被动听转录 → LLM 抽记忆/事件/提醒 → 蒸馏数字分身（人格档案）→ 反幻觉脚本复查 → FastAPI 控制端 + 安卓/Web 前端。设计上有多个值得称道的硬约束：时间戳=记录时间不冒充墙钟、日历 `when_dt` 只用 `temporal.resolve` 确定性规则（禁 LLM 编造日期）、`verify.run_all` 每次 ingest 后自动跑并删不落地项、`/inbox/upload` 已做路径遍历三重防御（W-006）、前端 `api.js` 固化 PA_BASE 防 SSRF（W-011）、`dev-server.js` 反代做了 Content-Type 校验 + 1MB body cap + nosniff（OD-001/OD-002）、LLM 错误体 `_sanitize_error_body` 脱敏 + api_key TTL 4h。

但深度审计发现 **4 HIGH / 3 MEDIUM / 3 LOW** 共 10 个问题，根因集中在三处结构性缺口：

1. **控制端零认证 + CORS `*` + 默认 `0.0.0.0`**：FastAPI app 无任何认证中间件，所有端点（含写/破坏性：POST /settings/llm、POST /verify 删数据、DELETE /chat-log、POST /ingest、POST /distill、POST /chat）裸暴露；CORS `allow_origins=["*"]` + `allow_methods=["*"]` + `allow_headers=["*"]`（api.py:15-16）；`cli.py serve` 默认 `--host 0.0.0.0`（cli.py:190）。三者叠加 → 任何能到达 8000 端口的主体（含用户访问的任意网页跨域 fetch）可读全部转录/记忆/人格档案、改 LLM base_url、删数据。这是本审计的头号风险。
2. **/settings/llm 任意 base_url → SSRF + PII + 凭证外泄**：POST /settings/llm 接受 `base_url` 无 scheme/host 校验（api.py:45,248-268）。攻击者经 #1 设 base_url=attacker.com → 此后每次 LLM/chat/distill/memory 调用把完整 prompt（含用户转录 PII + 人格档案）+ `Authorization: Bearer`/`x-api-key`（llm.py:420,449）发到攻击者服务器。`_post_json` 还无响应大小 cap（OOM）且允许 http://（凭证明文上链）。
3. **间接 prompt 注入 + 反幻觉 bigram 接地可绕过**：转录文本（来自 /inbox/upload，不可信）裸插进所有抽取/蒸馏/对话/日历/提醒 prompt（memory.py:28、distill.py:90、chat.py:29、calendar.py:42、reminders.py:67、recommend.py:32、wiki.py:68）。`verify.check_memories` 的内容溯源只要求 2 字 bigram 命中源文本（verify.py:159-160）——注入记忆只要含源段任一 2 字子串（如通用 bigram "的"/"我"）即过反幻觉落地。`verify` 只保证*可溯源性*，不保证*意图完整性*：注入可产出"落地但被操纵"的记忆/人格，进而污染 /chat 的"数字分身"回复。

**修复路线**：4 个 HIGH 中，PA-API-02（CORS）+ PA-API-03（base_url 校验）+ PA-LLM-01（_post_json size cap/scheme）+ PA-UPL-01（upload size cap）+ PA-STOR-01（原子写）为 1-3 文件 route-fix（下轮起逐个派生）；PA-API-01（零认证框架）+ PA-PII-01（prompt 注入防御）为 structural（写入 structural-debt.md，需更大授权单位）。3 LOW 中 PA-WEB-01 为 route-fix，PA-VER-01/PA-ID-01 为 info 不派生。

## 正面发现（已做好的硬约束，不派生 fix）

- 前端 api.js PA_BASE 固化 + scheme 校验（W-011）；bootstrap 不自动 POST /verify（W-012，最小权限）。
- dev-server.js 反代：nosniff 全局 + 方法守卫 + Content-Type 校验拒 non-json（OD-001）+ 1MB body cap（OD-002）+ 路径穿越 normalize+join + 端口范围守卫 + 优雅关闭——前端静态层是本次审计最硬的部分。
