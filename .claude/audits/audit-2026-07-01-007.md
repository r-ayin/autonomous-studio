## Findings

### AS-H-001 [HIGH] autonomous-loop.sh L124: while 循环内 cd 无 subshell 隔离
- **文件**: scripts/autonomous-loop.sh:124
- **严重度**: HIGH
- **类型**: route-fix
- **描述**: `cd "$ENGINE_DIR"` 在 while 循环体内直接执行，run_round 内部的 claude -p 或后续命令若改 cwd（如 cd 到子项目），下一轮 STOP_MARKER/case 路径基于错误 cwd 计算→引擎静默写入错误位置或找不到停止标记。
- **修复**: 把循环体包进 `( cd "$ENGINE_DIR" && ... )` subshell，或用 `pushd/popd`，或在 run_round 内显式 `cd "$ENGINE_DIR"` 前置。
- **优先级**: 1 (下轮派生 fix)

### AS-M-001 [MEDIUM] studio-lint-guard.sh: shell 转义断裂致 lint 静默失效 ✅ 已修
- **文件**: hooks/studio-lint-guard.sh:38,56
- **严重度**: MEDIUM
- **类型**: route-fix
- **描述**: 旧实现用 `echo "$INPUT" | python3 -c "...get('content')..."` 链式抽字段，content 含换行/引号/shell 元字符时 shell 转义断裂→lint 误判或静默放行语法错误。实测含 `\n` 和引号的 Python content 完全绕过校验。
- **修复**: 改用 `_AS_HOOK_INPUT="$(cat)"` 缓存 stdin 到环境变量，python3 -c 从 env 读完整 JSON，在 python 内完成所有字段抽取+语法校验+JSON 输出。shell 层只做 HOOK_EVENT 早筛，不经手 content 字符串。
- **验证**: 6 个测试用例全过（bad py block / bad json block / good py pass / valid json pass / non-Write pass / non-PreToolUse pass）
- **状态**: merged (case-2026-07-01-631 → optimization worktree)

### AS-M-002 [MEDIUM] distill-patterns.py L338-345: LLM 超时/拒答不可区分
- **文件**: scripts/distill-patterns.py:338-345
- **严重度**: MEDIUM
- **类型**: route-fix
- **描述**: step2_llm_distill subprocess.run timeout=120s 后 op=None 仍写 proposals.json，无法区分"LLM 拒答"(返回空)与"超时"(TimeoutExpired)。下游 agent 审提案时看到 null llm_output 不知是超时还是拒绝。
- **修复**: catch subprocess.TimeoutExpired 单独标 `"llm_status":"timeout"`；正常返回空标 `"llm_status":"empty_response"`；成功标 `"llm_status":"ok"`。

### AS-M-003 [MEDIUM] scout-scan.py L417-434: git diff --exit-code 大文件列表 E2BIG 风险
- **文件**: scripts/scout-scan.py:417-434
- **严重度**: MEDIUM
- **类型**: route-fix
- **描述**: count_pending_worktrees 内容等价对调 `git diff --exit-code main HEAD -- *touched`，当 touched>100 文件时命令行可能超 ARG_MAX(E2BIG)。且 exit-code 不区分"内容相同"与"仅空白差异"。
- **修复**: 改为逐文件 diff 或分批；或用 `git diff --stat` + 行数=0 判定等价。

### AS-L-001 [LOW] studio-progress-check.sh L5-12: heredoc python 变量注入风险
- **文件**: hooks/studio-progress-check.sh:5-12
- **严重度**: LOW
- **类型**: route-fix
- **描述**: `$CLAUDE_TOOL_INPUT` 经 heredoc python 解析，FILE_PATH 含特殊字符(引号/反斜杠)可破 PYEOF 边界或注入 python 代码。当前实际触发面窄（仅 prd.json 写入），但防御纵深不足。
- **修复**: 同 AS-M-001 模式，改 env var 传值。

### AS-L-002 [LOW] opt-worktree.sh L706-712: push stderr 被 sed 前缀化
- **文件**: scripts/opt-worktree.sh:706-712
- **严重度**: LOW
- **类型**: route-fix
- **描述**: `git push ... 2>&1 | sed 's/^/[push] /'` 把 stderr 也吃进前缀化，真错误信息(如 auth fail)被 `[push] ` 前缀污染难 grep。
- **修复**: 只 pipe stdout，stderr 单独处理或 tee。
