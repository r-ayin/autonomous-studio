## Findings

### ESE-TP-001 [HIGH] Shell Injection via unsanitized root path in subprocess

- **file**: 分诊流水线/scripts/pipeline-gate.py
- **line**: 80, 85
- **severity**: high
- **kind**: route-fix
- **finding**: `_diff_scale()` uses f-string interpolation into shell command: `f"git -C {root} diff --cached --numstat"` with `shell=True`. The `root` variable comes from `_project_root()` which resolves user-supplied file paths. A malicious or malformed workspace path containing shell metacharacters (e.g., `; rm -rf /`) would be executed verbatim.
- **remediation**: Replace `shell=True` + f-string with `subprocess.run(["git", "-C", str(root), "diff", "--cached", "--numstat"], shell=False, ...)`. Same fix for line 85 fallback.
- **evidence**: Lines 79-86 use `shell=True` with interpolated `{root}`; no sanitization applied.

### ESE-TP-002 [HIGH] Path traversal in history archive allows arbitrary file overwrite

- **file**: 分诊流水线/scripts/triage.py
- **line**: 88-93
- **severity**: high
- **kind**: route-fix
- **finding**: When `--done` is invoked, the archive filename is built from `d.get('kind', 'x')` which originates from user-supplied `--kind` argument stored in current.json. Although `--kind` has `choices=["small", "complex"]` validation at CLI parse time, if current.json is manually edited or corrupted to contain a kind like `"../../etc/cron.d/backdoor"`, the archive write `(hist / f"{n:04d}-{d.get('kind', 'x')}.json").write_text(...)` will traverse out of `.pipeline/history/` and overwrite arbitrary files.
- **remediation**: Sanitize `kind` before use in filename: `safe_kind = re.sub(r'[^a-z]', '', d.get('kind', 'x')) or 'unknown'`. Or validate `kind in STAGES_KINDS` before constructing path.
- **evidence**: Line 91 constructs path from unvalidated dict value; no check that resulting path stays within `hist/`.

### ESE-TP-003 [MEDIUM] Silent exception swallowing hides security-relevant failures

- **file**: 分诊流水线/scripts/pipeline-gate.py
- **line**: 50-51, 95-96, 161-162
- **severity**: medium
- **kind**: route-fix
- **finding**: Multiple bare `except Exception: return None` / `pass` blocks silently swallow errors. In a security gate, silent failure = open door. If JSON parsing fails due to corruption or tampering, the gate returns `None` and subsequent checks may pass when they should block. Line 161-162 in Stop handler swallows ALL exceptions including permission errors.
- **remediation**: At minimum log to stderr. For `_load_current`, distinguish "file missing" (return None OK) from "file corrupt" (should warn or block). For `_diff_scale`, log git failures. For Stop handler, at least catch specific exceptions.
- **evidence**: Three separate bare except blocks that silently discard errors in a security-critical context.

### ESE-TP-004 [MEDIUM] Race condition in history numbering causes data loss

- **file**: 分诊流水线/scripts/triage.py
- **line**: 90
- **severity**: medium
- **kind**: route-fix
- **finding**: History archive number computed as `len(list(hist.glob("*.json"))) + 1`. Under concurrent invocations (parallel agents), two processes can read same count N and both write to `{N+1:04d}-*.json`, with the second overwriting the first's archive. Data loss.
- **remediation**: Use atomic write pattern: generate unique name (timestamp + pid), write to temp, rename. Or use `O_CREAT|O_EXCL` flag. Simplest: include ISO timestamp in filename instead of sequential number.
- **evidence**: TOCTOU between glob count and file write on line 90-92.

### ESE-TP-005 [MEDIUM] Stage regression check uses string comparison, not enum safety

- **file**: 分诊流水线/scripts/triage.py
- **line**: 118
- **severity**: medium
- **kind**: route-fix
- **finding**: `STAGES.index(args.stage) < STAGES.index(d.get("stage", "requirement"))` will raise `ValueError` if `d["stage"]` contains a value not in STAGES list (e.g., from manual JSON edit). This crashes with unhandled exception instead of giving clear error. Also, default "requirement" masks corruption.
- **remediation**: Validate `d["stage"] in STAGES` before index lookup; if invalid, treat as corrupt and either reset or error clearly.
- **evidence**: No validation of stored stage value against STAGES enum before `.index()` call.

### ESE-TP-006 [LOW] DOCS allowlist missing .claude/ directory despite SKILL.md claiming exemption

- **file**: 分诊流水线/scripts/pipeline-gate.py
- **line**: 27-28, 64-73
- **severity**: low
- **kind**: route-fix
- **finding**: SKILL.md states "只改 `.md` / `.pipeline/` / `.claude/` 不拦" but `_is_doc_or_meta()` only checks DOCS set (individual filenames) and `planning/` + `.pipeline/` prefixes. `.claude/` directory is NOT exempted — editing `.claude/settings.json` would trigger the gate despite docs saying it shouldn't.
- **remediation**: Add `.claude` prefix check in `_is_doc_or_meta()`: `rel == ".claude" or rel.startswith(".claude" + os.sep)`.
- **evidence**: SKILL.md line 54 vs code lines 64-73 mismatch.
