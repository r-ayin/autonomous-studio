## Context

wanxia-edit 是 sunset-prediction (已审 audit-2026-07-06-020) 的姊妹项目/编辑副本，用于晚霞预测内容生成+社媒数据采集+邮件推送。与 sunset-prediction 共享部分逻辑但独立部署（引用 /opt/wanxia/ 路径）。代码混合 ESM/CJS/Python，依赖 Playwright + 微博/XHS API + DeepSeek/GPT-Image-2 API。

## Findings

### HIGH

#### H-001: Cross-project credential coupling — hardcoded /opt/personal-assistant/.env
- **File**: `generate-content.js:95`
- **Severity**: high
- **Kind**: route-fix
- **Finding**: `readFileSync('/opt/personal-assistant/.env', 'utf8')` 硬编码读取另一个项目的 .env 文件获取 DEEPSEEK_API_KEY。跨项目凭证耦合：personal-assistant 改路径/权限即断；无 stat mode 校验（任何读权限用户可窃 key）；失败时静默回退原 headline（安全但掩盖配置错误）。
- **Remediation**: 改用本项目的 `.env` 或 `process.env.DEEPSEEK_API_KEY`；若必须跨项目读，加 fs.statSync().mode & 0o077 === 0 权限检查 + 显式错误日志。

#### H-002: Same cross-project credential coupling in Python
- **File**: `poetic_intro.py:20`
- **Severity**: high
- **Kind**: route-fix
- **Finding**: `_load_key()` 硬编码 `for p in ("/opt/personal-assistant/.env",)` 读取外部项目凭证。同 H-001 风险。bare `except: pass` 吞所有异常包括权限错误。
- **Remediation**: 同 H-001。优先 `os.environ.get("DEEPSEEK_API_KEY")`；fallback 读本目录 .env；加权限校验。

#### H-003: Hardcoded absolute paths to /opt/wanxia/ — non-portable + implicit deployment dependency
- **File**: `dl_images.mjs:5,10-12`, `analyze.mjs:8,13-15`
- **Severity**: high
- **Kind**: route-fix
- **Finding**: 两个分析脚本硬编码 `/opt/wanxia/.env`、`/opt/wanxia/src/cities.js`、`/opt/wanxia/node_modules/better-sqlite3`、`/opt/wanxia/data/sunset.db`。在 workspace 副本里跑必 fail；隐式绑定生产部署路径；DB readonly 模式安全但路径泄漏部署拓扑。
- **Remediation**: 改为相对路径 + import.meta.url 解析；或从 WANXIA_ROOT env var 读基址；DB 路径也走 env。

#### H-004: Playwright --no-sandbox + unvalidated proxy URL
- **File**: `social-scraper.js:30,42-44`
- **Severity**: high
- **Kind**: route-fix
- **Finding**: Chromium launch args 含 `--no-sandbox`（容器环境常见但削弱隔离）；SOCIAL_PROXY_URL 直接从 env 注入 Playwright proxy.server 无 scheme/IP 校验。若 env 被污染（如 SSRF 测试残留 `http://169.254.169.254/...`），浏览器流量经恶意代理→Cookie 泄露。
- **Remediation**: 校验 SOCIAL_PROXY_URL 格式（仅 http/https/socks5，IP 非 link-local/metadata）；--no-sandbox 仅在 CI/container 检测到时启用。

#### H-005: shell spawn with cwd from URL pathname
- **File**: `server.js:78`
- **Severity**: medium (downgraded from high — command is literal, no user input)
- **Kind**: route-fix
- **Finding**: `execAsync('node scripts/generate-content.js', { cwd })` 中 cwd = `new URL('.', import.meta.url).pathname`。命令字面量安全（无注入向量），但若 import.meta.url 含特殊字符（空格/引号/shell metachar）cwd 可能异常导致 spawn 失败或在意外目录执行。实际风险低（ESM URL 编码保证合法路径），标记为防御性改进。
- **Remediation**: 用 `fileURLToPath()` 转换（已在 line 28 import 但未用在此处）；或改用 execFileSync 避免 shell。

### MEDIUM

#### M-001: Mail attachment unbounded when PIL missing
- **File**: `mail_push.py:140-156`
- **Severity**: medium
- **Kind**: route-fix
- **Finding**: PIL 可用时转 JPG ~300KB/张；PIL 缺失时直接 `cf.read_bytes()` 挂原 PNG（注释说 2.3MB/张，8 张 = 18MB base64 后 ~24MB）。SMTP timeout=300s 可能够但边界；大附件触发收件方拒收/OOM。无单文件大小上限。
- **Remediation**: 加 `MAX_ATTACHMENT_BYTES = 5 * 1024 * 1024`；超限跳过 + warn；PIL 缺失时也 cap。

#### M-002: Weibo desktop fetch unbounded response body
- **File**: `social-scraper.js:192`
- **Severity**: medium
- **Kind**: route-fix
- **Finding**: `const html = await res.text()` 无大小限制。weibo 搜索页可达数 MB；异常响应（重定向到登录页带大量 JS）更大。fetchMobile 走 JSON API 较小，但 desktop HTML 应 cap。
- **Remediation**: 用 ReadableStream + AbortController 限 2MB；或 res.arrayBuffer() 后判 byteLength。

#### M-003: Duplicate .env loading with silent override
- **File**: `server.js:6-14` + `env.js:22-45`
- **Severity**: medium
- **Kind**: route-fix
- **Finding**: server.js 顶层读当前目录 .env 并写入 process.env；然后第一行 `import './src/env.js'`（实际是 ./env.js？注释说 src/env.js 但文件在根）又读一次。后者静默覆盖前者。注释承认 WANXIA-DEAD-01 structural debt。两份解析器行为微差（env.js 支持 WANXIA_ENV_PATH override + 多候选路径）。
- **Remediation**: 删 server.js:6-14 冗余加载；只保留 env.js 单一入口。

#### M-004: Vision calibrate silent failure masking
- **File**: `vision_calibrate.py:40-42`
- **Severity**: medium
- **Kind**: route-fix
- **Finding**: classify() 三次重试全失败返回 `{"is_sunset": False, "vividness": 0, "reason": "fail"}`。API 不可达/token 过期/网络抖动均被当作"非晚霞"计入统计，拉低相关性系数。无 stderr 日志区分真负 vs 错误。
- **Remediation**: 失败时 reason 写具体异常类型；results 加 `error: true` 字段；聚合统计时排除 error 条目。

#### M-005: Non-atomic write in analyze.mjs
- **File**: `analyze.mjs:114`
- **Severity**: medium
- **Kind**: route-fix
- **Finding**: `writeFileSync("/tmp/social_analysis.json", ...)` 非原子；crash 时留半截 JSON；/tmp world-readable 泄漏分析数据（含微博帖子正文片段）。
- **Remediation**: tempfile + os.replace 原子写；输出到项目内 data/ 而非 /tmp；chmod 0600。

#### M-006: Image download silent discard + non-atomic write
- **File**: `dl_images.mjs:46-53`
- **Severity**: medium
- **Kind**: route-fix
- **Finding**: `if (buf.length < 5000) continue` 丢弃小图无日志（可能是占位符/错误页，值得记录）；`writeFileSync(fp, buf)` 非原子；并发跑多实例可能写坏文件。
- **Remediation**: 丢弃时 console.warn；tempfile + rename 原子写。

#### M-007: generate-content.js posts.json non-atomic write
- **File**: `generate-content.js:176`
- **Severity**: medium
- **Kind**: route-fix
- **Finding**: `writeFileSync(join(OUT_DIR, 'posts.json'), ...)` 非原子。cron 中途 crash 留损坏 JSON → 下游 mail_push/generate-cover-image 解析失败。
- **Remediation**: fs.writeFileSync(tmp) + fs.renameSync(tmp, target)。Node 18+ 可用 fs.promises.writeFile + rename。

### LOW

#### L-001: Empty catch blocks swallow errors silently
- **File**: `social-scraper.js:226`, `env.js:43`, `generate-content.js:37`
- **Severity**: low
- **Kind**: route-fix
- **Finding**: 多处 `catch {}` 或 `catch { /* skip */ }` 吞掉所有异常。调试困难；配置错误表现为"功能静默不工作"而非明确报错。
- **Remediation**: 至少 `catch(e) { console.debug(...) }`；关键路径（env 加载、API 调用）升级为 warn/error。

#### L-002: Copy-generator hardcoded city landmarks + color variants
- **File**: `copy-generator.js:19-54, 85-120+`

- **Finding**: SPOTS/COLOR_VARIANTS 大量硬编码中文文案。新增城市需改源码；文案调整需重新部署。适合抽配置文件。
- **Remediation**: 抽 spots.json + color-variants.json；运行时 load。标 structural-debt。

#### L-003: validate_new.js references stale /opt/wanxia paths

- **Remediation**: 同 H-003 统一处理。可与 H-003 合并为一个 fix。
