# AS-M-001: lint-guard env var JSON safety

- **Source commit**: a6d787a71250ab34cd1401773a76db6a6842c70a

- **File changed**: hooks/studio-lint-guard.sh (66 insertions, 58 deletions)

## Motivation

studio-lint-guard.sh passed hook input (JSON) via shell arguments. Complex JSON
with special characters (quotes, backslashes, newlines) would break shell
expansion, causing the lint guard to silently pass when it should have caught
violations. The fix switches to environment variable passing which avoids
shell escaping issues entirely.

## Why not merge the whole branch

The `auto/optimization` standing worktree is 9 commits ahead of main. Its base
is stale -- it predates at least 3 safety guards merged into main since
(commit-gate reset-bypass fix, merge branch-delete auto/*, ensure_main_wt
fast-forward). Squash-merging the entire branch would regress those guards.
Only the M-001 and M-002 commits contain new value; the rest are state-sync
churn (scout snapshots, rebase-resolve, audit-cycle-state writes).

## Re-apply plan

Cherry-pick or `git apply` this patch onto current main in a fresh worktree.
Verify hooks/studio-lint-guard.sh still applies cleanly, then run the hook
against a test JSON payload with special characters.
