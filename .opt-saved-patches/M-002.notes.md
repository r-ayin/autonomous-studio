# AS-M-002: distill LLM status differentiation

## Motivation

distill-patterns.py `step2_llm_distill` treated all LLM failures uniformly:
timeout, empty response, nonzero exit, and exceptions all resulted in
`op=None` being silently swallowed. This made it impossible to diagnose
whether distillation failed because the LLM was overloaded (timeout), returned
garbage (empty), crashed (nonzero exit), or hit an unexpected error
(exception). The fix adds a `llm_status` field to proposals distinguishing
these four failure modes.

state-sync churn commits that would regress 3 safety guards already in main.

Cherry-pick or `git apply` onto current main. The change is confined to

`llm_status` field appears in the output proposals.
