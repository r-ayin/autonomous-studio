# 简历筛选技能

基于历史招聘数据（157份简历，118通过/38不通过）训练出的筛选规则，用于AI购物助手团队（淘天）的标注人员和管理岗招聘。

## 岗位概览

团队招聘四类岗位：
1. **执行岗**（标注/评测）— 通过率最高(96%)，门槛相对低
2. **组长岗** — 通过率50%，需要带队经验+评测深度
3. **高阶管理**（AI产品经理等）— 通过率56%，要求行业深度+顶层设计能力
4. **食品生鲜知识库运营** — 独立品类，需要食品行业+电商经验

## 筛选流程

收到简历后按以下步骤执行：

### Step 1: 提取简历文本
如果是PDF文件，用 `scripts/extract_pdf.py` 提取文本：
```bash
python3 <skill-path>/scripts/extract_pdf.py <pdf_path>
```

### Step 2: 判断目标岗位
根据简历内容和用户指定的岗位，确定属于哪类岗位（执行/组长/高阶管理/食品生鲜）。

### Step 3: 按岗位规则评估
读取 `references/screening_rules.md` 中对应岗位的详细规则进行评估。

### Step 4: 输出评估报告
使用以下格式输出：

```
## 简历评估报告

**候选人**: [姓名]
**目标岗位**: [岗位名称]
**岗位类型**: [执行/组长/高阶管理/食品生鲜]

### 评估结论: ✅ 建议通过 / ❌ 建议不通过

### 匹配度评分: [1-10]

### 正面信号
- [列出简历中的正面关键词和经历]

### 风险信号
- [列出可能导致不通过的因素]

### 不通过原因（如适用）
[参考历史不通过原因模板]

### 面试建议
[基于历史面试反馈，建议面试时重点考察的方向]
```

### Step 5: 提供看板链接
筛选完成后，通过 port-mapping 获取看板的公网链接并展示给用户：
```bash
curl -s "http://localhost:58596/api/port-mapping?port=8899" | jq -r .url
```
在评估报告末尾附上：
```
📊 查看筛选看板: [上面获取到的URL]
```
如果 port-mapping 请求失败，跳过此步骤即可。

## 关键筛选维度

从历史数据中提炼的核心评估维度（按重要性排序）：

### 最强正面信号（通过组显著高于不通过组）
1. **用户视角** (+25.6%) — 能从用户角度思考产品体验，有用户需求洞察
2. **搜索评测经验** (+26.8%) — 有搜索引擎/搜索结果评测经验
3. **导购/推荐经验** (+20.7%) — 有商品推荐、种草、导购相关经验
4. **AI/大模型经验** (+11.0%) — 有AI训练、大模型评测、Prompt工程经验

### 最强负面信号（不通过组显著高于通过组）
1. **纯管理背景** (-28.2%) — 只有团队管理经验但缺乏一线评测/标注能力
2. **badcase仅停留在理论** (-17.0%) — 提到badcase但没有电商场景实操
3. **纯客服背景** (-11.6%) — 仅有客服经验缺少评测/标注转化
4. **食品行业无关经验** (-11.7%) — 非食品电商的食品经验（如食品研发）
5. **纯运营/直播背景** (-10.0%) — 偏GMV/直播运营，缺少用户体验视角

### 历史不通过原因TOP（排除流程原因后）
1. **缺少用户视角** (22次) — 最常见原因
2. **缺少电商平台评测经验** (21次)
3. **无AI相关项目经验** (6次)
4. **经验面过窄/单一领域** (6次)
5. **食品生鲜行业不匹配** (7次)
